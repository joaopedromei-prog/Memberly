## Story

**As a** admin,
**I want** gerar banners via IA diretamente nos formulários de produto e módulo,
**so that** eu não precise de um wizard separado para obter banners visuais.

---

## Acceptance Criteria

1. Nos formulários de criação/edição de produto, botão "Gerar Banner com IA" visível ao lado do campo de upload de banner
2. Nos formulários de criação/edição de módulo, botão "Gerar Banner com IA" com mesma funcionalidade
3. Ao clicar, o sistema envia o título e descrição do produto/módulo para a Gemini API via `/api/ai/generate-banner`
4. Loading state exibido durante geração com mensagem "Gerando banner..." e spinner/animação
5. Banner gerado exibido como preview para o admin aprovar
6. Admin pode: aceitar (usa o banner gerado), regenerar (nova geração), ou cancelar (voltar ao upload manual)
7. Banner aceito é automaticamente salvo no Supabase Storage e o URL é preenchido no formulário
8. Se a Gemini API falhar, exibir mensagem amigável com opção de retry ou fallback para upload manual
9. O botão de geração só aparece quando título do produto/módulo está preenchido (validação)

## Dev Notes

### Fluxo de Geração

```
Admin preenche título/descrição → Clica "Gerar Banner com IA"
→ POST /api/ai/generate-banner { title, description }
→ Gemini API gera imagem → base64 → upload Supabase Storage → URL
→ Preview no formulário → Admin aceita → URL preenchido no campo banner_url
```

### Considerações

- Reutilizar a infraestrutura existente de `gemini-client.ts` e `/api/ai/generate-banner`
- O `BannerGenerator` deve ser um componente standalone reutilizável
- O componente `ImageUpload` existente deve continuar funcionando como alternativa manual

## Story

**As a** developer/DBA,
**I want** indexes otimizados para queries de ordenação e colunas `updated_at` para auditoria,
**so that** queries de listagem com ordenação sejam performáticas e eu saiba quando cada registro foi atualizado.

---

## Context

### Estado Atual

O Brownfield Assessment identificou gaps no schema:

**Indexes faltando (5):**
- `modules(product_id, sort_order)` — queries de listagem de módulos por produto
- `lessons(module_id, sort_order)` — queries de listagem de lições por módulo
- `comments(lesson_id, created_at DESC)` — listagem de comentários ordenados
- `lesson_progress(lesson_id)` — agregações de progresso por lição
- `lesson_bookmarks(lesson_id)` — lookup de bookmarks

**Colunas `updated_at` faltando (5 tabelas):**
- `modules`, `lessons`, `comments`, `lesson_progress`, `member_access`

### Mudança Necessária

Criar uma nova migration que:
1. Adiciona 5 indexes compostos
2. Adiciona coluna `updated_at TIMESTAMPTZ DEFAULT NOW()` nas 5 tabelas
3. Cria triggers `BEFORE UPDATE` para auto-atualizar `updated_at`
4. Backfill `updated_at` com `created_at` para registros existentes

### Decisões Técnicas

- Migration única e aditiva (sem DROP, sem ALTER TYPE)
- Indexes criados com `IF NOT EXISTS` para idempotência
- Triggers seguem o padrão existente de `profiles` e `products`
- `CREATE INDEX CONCURRENTLY` não usado (requer fora de transaction, desnecessário no volume atual)

### SQL Esperado (Referência)

```sql
-- Composite indexes
CREATE INDEX IF NOT EXISTS idx_modules_product_sort ON modules(product_id, sort_order);
CREATE INDEX IF NOT EXISTS idx_lessons_module_sort ON lessons(module_id, sort_order);
CREATE INDEX IF NOT EXISTS idx_comments_lesson_created ON comments(lesson_id, created_at DESC);
CREATE INDEX IF NOT EXISTS idx_lesson_progress_lesson ON lesson_progress(lesson_id);
CREATE INDEX IF NOT EXISTS idx_lesson_bookmarks_lesson ON lesson_bookmarks(lesson_id);

-- updated_at columns + triggers (repeat for each table)
ALTER TABLE modules ADD COLUMN IF NOT EXISTS updated_at TIMESTAMPTZ DEFAULT NOW();
UPDATE modules SET updated_at = created_at WHERE updated_at IS NULL;
CREATE TRIGGER set_modules_updated_at BEFORE UPDATE ON modules
  FOR EACH ROW EXECUTE FUNCTION update_updated_at_column();
```

## Story

**Como** membro da plataforma,
**Quero** poder fazer login usando meu numero de telefone (recebendo um codigo SMS),
**Para que** eu tenha uma alternativa de acesso mesmo sem verificar meu email.

## Acceptance Criteria

1. **AC1** — Feature flag: tab de login por telefone so e exibida quando `NEXT_PUBLIC_PHONE_AUTH_ENABLED=true` (default: `false`).
2. **AC2** — Pagina de login exibe opcao de alternar entre "Email" e "Telefone" (tabs ou toggle) quando feature flag ativa.
3. **AC3** — Ao selecionar telefone, campo de input com mascara de telefone brasileiro (+55) e exibido.
4. **AC4** — Antes de enviar OTP, sistema faz lookup em `profiles.phone` via API route server-side. Se telefone nao encontrado, exibe erro: "Telefone nao encontrado. Verifique o numero ou use seu email." SEM chamar `signInWithOtp`.
5. **AC5** — Somente apos confirmar que o telefone existe em `profiles`, sistema chama `supabase.auth.signInWithOtp({ phone })` para enviar SMS.
6. **AC6** — Tela de verificacao exibida apos envio: campo para digitar codigo de 6 digitos com auto-focus e countdown de reenvio (60s).
7. **AC7** — Ao verificar codigo correto via `supabase.auth.verifyOtp({ phone, token, type: 'sms' })`, usuario e autenticado e redirecionado por role.
8. **AC8** — Rate limiting de SMS: maximo 3 envios por telefone a cada 10 minutos, maximo 10 por dia por telefone. Apos exceder, exibir erro amigavel.
9. **AC9** — Supabase Auth Phone provider deve estar configurado manualmente no Dashboard (Twilio ou equivalente). Documentar passos em `docs/guides/phone-auth-setup.md`.
10. **AC10** — Variaveis `NEXT_PUBLIC_PHONE_AUTH_ENABLED` e instrucoes de setup Twilio adicionadas ao `.env.example`.
11. **AC11** — Testes unitarios para o fluxo de login por telefone (5 cases: envio ok, telefone invalido, codigo errado, telefone nao cadastrado, rate limit excedido).

## Dev Notes

### Fluxo corrigido (lookup ANTES do OTP)

```
Usuario digita telefone
  -> POST /api/auth/phone-lookup (server-side, admin client)
  -> Se NAO encontrado: erro "Telefone nao encontrado"
  -> Se encontrado: signInWithOtp({ phone })
  -> SMS enviado -> Tela de verificacao OTP
  -> verifyOtp -> Autenticado -> Redirect por role
```

### Supabase Phone Auth

```typescript
// Enviar OTP (somente apos confirmar lookup)
const { error } = await supabase.auth.signInWithOtp({
  phone: '+5511999999999',
});

// Verificar OTP
const { data, error } = await supabase.auth.verifyOtp({
  phone: '+5511999999999',
  token: '123456',
  type: 'sms',
});
```

### Pre-requisitos manuais (NAO automatizaveis)

1. Supabase Dashboard > Authentication > Providers > Phone > Enable
2. Configurar Twilio: Account SID, Auth Token, Messaging Service SID
3. Custo: ~R$0.05-0.10 por SMS enviado (Twilio)

### Rate Limiting

Usar storage em memoria (Map) com limpeza periodica. Para producao, considerar Redis.

```typescript
const smsRateLimit = new Map<string, { count: number; resetAt: number; dailyCount: number; dailyResetAt: number }>();
```

### Dependencias

- **Story 17.4** — Telefone deve estar salvo no perfil para o lookup funcionar
- **Story 17.2** — Login por email via magic link ja implementado (tabs compartilham a mesma pagina)

### Mascara de telefone

Usar formato brasileiro: `+55 (11) 99999-9999`. Considerar lib `react-input-mask` ou implementacao manual.

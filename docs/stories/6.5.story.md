## Story

**As a** admin,
**I want** visualizar o produto exatamente como o membro verá,
**so that** eu possa validar a experiência antes de publicar.

---

## Acceptance Criteria

1. Botão "Preview como Membro" na página de edição de produto no admin
2. Ao clicar, abre uma nova aba/janela com a visualização do produto no layout Netflix-like da área de membros
3. A preview exibe: banner hero do produto, lista de módulos com banners, aulas de cada módulo — no tema escuro de membro
4. A preview mostra o produto mesmo que não esteja publicado (bypass de `is_published` para preview)
5. A preview mostra todas as aulas, incluindo rascunhos, com indicador visual de "rascunho" (badge)
6. A preview inclui um banner fixo no topo indicando "Modo Preview — Esta é a visão do membro"
7. O admin não pode interagir como membro na preview (comentários desabilitados, progresso não salvo)
8. URL da preview segue padrão: `/admin/products/[id]/preview`
9. A preview é responsiva (funciona em desktop e mobile)

## Dev Notes

### Abordagem

- Reutilizar os componentes da área de membros (Epic 3) para renderizar a preview
- Usar uma flag `isPreview` para desabilitar interações e mostrar indicadores de rascunho
- A página de preview é uma rota admin (`/admin/products/[id]/preview`) mas usa o layout visual de membro

### Considerações

- A preview deve carregar dados diretamente do servidor (RSC) sem depender de `member_access`
- Garantir que a URL de preview não seja acessível por membros (middleware de admin)

## Story

**As an** admin,
**I want** um botão "Adicionar membro" em verde com ícone de "+" ao lado do botão "Exportar" na página `/admin/members`,
**so that** eu possa cadastrar manualmente um novo membro diretamente pelo painel admin sem precisar importar CSV.

---

## Context

### Estado Atual

A página `/admin/members` possui dois botões no header: "Importar CSV" e "Exportar". Para adicionar um único membro, o admin precisa criar um CSV com uma linha e usar o fluxo de importação — processo desnecessariamente complexo para uma operação simples.

A API `POST /api/members/import` já cria usuários via Supabase Auth (`admin.auth.createUser`), cria/atualiza profiles, concede acesso a produtos e envia email de recuperação de senha. Essa lógica pode ser reutilizada para criação individual.

### Mudança Necessária

1. Adicionar botão verde "Adicionar membro" com ícone `Plus` (lucide-react) ao lado do botão "Exportar" no `MemberList.tsx`
2. Criar componente `AddMemberDialog.tsx` — modal com formulário: nome completo, email, produto (opcional)
3. Criar endpoint `POST /api/members` — cria usuário individual via Supabase Auth + profile + acesso opcional a produto

### Decisões Técnicas

- Botão usa estilo verde (`bg-emerald-600 hover:bg-emerald-700 text-white`) para diferenciá-lo visualmente dos demais botões (branco/outline)
- Ícone `Plus` do lucide-react (consistente com o design system existente)
- O modal segue o padrão de `ImportMembersDialog` (dialog overlay com backdrop blur)
- Endpoint separado `POST /api/members` (ao invés de reutilizar `/import`) para manter semântica REST limpa
- Email de recuperação de senha é enviado automaticamente ao novo membro (mesmo padrão do import)

---

## Acceptance Criteria

1. Botão "Adicionar membro" aparece em verde (`bg-emerald-600`) com ícone `Plus` à esquerda do texto, posicionado ao lado do botão "Exportar" no header da página `/admin/members`
2. Ao clicar no botão, abre um modal/dialog com formulário contendo: **Nome completo** (obrigatório), **Email** (obrigatório, validação de formato), **Produto** (select opcional com lista de produtos disponíveis)
3. Ao submeter o formulário com dados válidos, o sistema cria o usuário no Supabase Auth, cria o profile e (se produto selecionado) concede acesso ao produto
4. Email de recuperação de senha é enviado automaticamente ao email cadastrado
5. Após criação bem-sucedida, exibe toast de sucesso e a lista de membros é atualizada (router.refresh)
6. Erros são tratados: email duplicado exibe mensagem amigável, campos obrigatórios mostram validação inline
7. Build, typecheck e lint passam sem erros

### API Existente (GET /api/members)

O arquivo `src/app/api/members/route.ts` já possui o handler GET. Adicionar o handler POST no mesmo arquivo.

**⚠️ Auth Guard (PO Review):** O handler POST DEVE replicar o mesmo padrão de verificação de admin auth usado no handler GET existente. Verificar o padrão no GET e aplicar identicamente no POST antes de qualquer operação.

### Padrão de Criação de Usuário (do Import)

```typescript
// Criar usuário no Supabase Auth
const { data: authData, error: authError } = await supabaseAdmin.auth.createUser({
  email,
  password: crypto.randomUUID(),
  email_confirm: true,
});

// Criar/atualizar profile
await supabaseAdmin.from('profiles').upsert({
  id: authData.user.id,
  full_name,
  role: 'member',
});

// Conceder acesso (se produto selecionado)
if (productId) {
  await supabaseAdmin.from('member_access').insert({
    profile_id: authData.user.id,
    product_id: productId,
    granted_by: 'manual',
  });
}

// Enviar email de recuperação
await supabaseAdmin.auth.resetPasswordForEmail(email);
```

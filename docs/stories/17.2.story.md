## Story

**Como** membro da plataforma,
**Quero** fazer login apenas informando meu email e recebendo um link magico,
**Para que** eu nao precise memorizar senhas e tenha acesso simples e seguro.

## Acceptance Criteria

1. **AC1** — Pagina de login (`/login`) exibe apenas campo de email (sem campo de senha).
2. **AC2** — Ao submeter o email, o sistema chama `supabase.auth.signInWithOtp({ email, options: { emailRedirectTo } })` para enviar magic link.
3. **AC3** — Mensagem de confirmacao exibida apos envio: "Enviamos um link de acesso para seu email. Verifique sua caixa de entrada."
4. **AC4** — Callback route `/auth/callback` configurada para processar o token do magic link e redirecionar baseado no role (admin -> `/admin`, member -> `/`).
5. **AC5** — Pagina `/forgot-password` removida ou redirecionada para `/login` (nao ha mais necessidade de reset de senha).
6. **AC6** — Middleware atualizado para incluir `/auth/callback` como rota publica.
7. **AC7** — UI do login mantém o design Netflix-like existente (tema escuro, animacoes), apenas removendo o campo de senha e ajustando o botao para "Enviar link de acesso".
8. **AC8** — Fallback: se o Supabase retornar erro no envio do OTP, exibir mensagem de erro amigavel com opcao de reenviar.
9. **AC9** — Testes unitarios para o componente de login atualizado (minimo 3 test cases: envio ok, erro de API, email invalido).

## Dev Notes

### Supabase Magic Link (OTP)

```typescript
const { error } = await supabase.auth.signInWithOtp({
  email,
  options: {
    emailRedirectTo: `${window.location.origin}/auth/callback`,
  },
});
```

### Auth Callback Handler

```typescript
// src/app/auth/callback/route.ts
import { createClient } from '@/lib/supabase/server';
import { NextRequest, NextResponse } from 'next/server';

export async function GET(request: NextRequest) {
  const { searchParams } = new URL(request.url);
  const code = searchParams.get('code');

  if (code) {
    const supabase = await createClient();
    await supabase.auth.exchangeCodeForSession(code);
    // Redirect based on role
  }
}
```

### Impacto

- `member-provisioning.ts` continua criando usuarios com senha default (necessario para Supabase Auth createUser), mas a senha nao sera mais usada pelo membro — ele sempre usa magic link.
- O campo de senha no Supabase Auth permanece como requirement interno, mas e transparente ao usuario.

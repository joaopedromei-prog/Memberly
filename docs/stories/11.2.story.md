## Dev Notes

### Context

O componente `@/components/ui/Input.tsx` já existe com suporte a `label`, `error` e estilização padrão, mas **nenhum formulário o utiliza**. Todos os forms admin e dialogs usam `<input>` raw com estilos duplicados.

Isso resulta em:
- Estilos de input ligeiramente diferentes entre forms
- Focus states inconsistentes (`focus:ring-blue-500` vs `focus:border-blue-500`)
- Labels com formatting variado
- Error handling reimplementado em cada form

### Decisões Técnicas

- Manter a API do `Input.tsx` compatível com `React.InputHTMLAttributes` (spread props)
- Criar `Textarea.tsx` e `Select.tsx` complementares seguindo mesmo padrão
- Não alterar comportamento de validação — apenas trocar o elemento renderizado
- O `RichTextEditor` (TipTap) em `LessonForm.tsx` NÃO é um input — não migrar
- File inputs (`<input type="file">`) em `ImportMembersDialog.tsx` NÃO migram para `Input.tsx`
- Migrar um form por vez para facilitar review

### Nota de Escopo

Esta story tem 14 arquivos para migrar. Se o escopo se mostrar excessivo durante execução, as Tasks 4-5 (settings + dialogs) podem ser movidas para uma story separada. Priorizar Tasks 1-3 (componentes base + forms principais).

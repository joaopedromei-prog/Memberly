## Story

**As an** admin,
**I want** uma página de configurações completa no painel admin,
**so that** eu possa customizar branding (logo, cores, nome da plataforma), gerenciar webhook secret, e configurar templates de email — tudo em um só lugar.

---

## Context

### Estado Atual

A página `/admin/settings` existe como placeholder com texto "Coming soon". O sidebar admin já tem o item "Configurações" apontando para essa rota. Não existe tabela de settings no banco — todas as configurações estão hardcoded ou inexistentes.

### Mudança Necessária

Implementar a página completa com 3 seções (Branding, Webhooks, Email), criar tabela `site_settings` no Supabase para persistência, e expor API routes para CRUD dessas configurações.

### Decisões Técnicas

- Tabela `site_settings` usa modelo key-value com valor JSONB (flexível para diferentes tipos de configuração)
- Upload de logo via Supabase Storage (bucket `branding`)
- Color picker usa input nativo `type="color"` (sem dependência extra)
- Template de email usa textarea simples (não rich text editor — evita dependência pesada para v1)

---

## Acceptance Criteria

1. Página `/admin/settings` renderiza com 3 seções em tabs: **Branding**, **Webhooks**, **Email**
2. **Branding**: campos para nome da plataforma (text input), logo (upload com preview), cor primária (color picker). Valores persistidos na tabela `site_settings`
3. **Webhooks**: exibe URL do webhook (read-only, com botão copiar), mostra secret mascarado (`••••••••`) com botão para revelar, botão para gerar/rotacionar secret (com confirmação)
4. **Email**: campo de template de boas-vindas (textarea com variáveis suportadas: `{{name}}`, `{{product}}`), preview renderizado ao lado
5. Feedback visual de sucesso/erro ao salvar cada seção (toast)
6. Tabela `site_settings` criada com RLS (somente admin lê/escreve)
7. Build, typecheck e lint passam sem erros

## Dev Notes

### Estrutura da tabela site_settings

```sql
CREATE TABLE site_settings (
  key TEXT PRIMARY KEY,
  value JSONB NOT NULL DEFAULT '{}',
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- Seeds
INSERT INTO site_settings (key, value) VALUES
  ('platform_name', '"Memberly"'),
  ('primary_color', '"#3B82F6"'),
  ('logo_url', 'null'),
  ('webhook_secret', to_jsonb(gen_random_uuid()::text)),
  ('welcome_email_template', '"Olá {{name}}, bem-vindo ao curso {{product}}!"');
```

### Webhook URL

A URL do webhook é construída dinamicamente: `{window.location.origin}/api/webhooks/payt` (read-only, não é persistida no banco).

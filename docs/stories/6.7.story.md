## Story

**As a** admin,
**I want** fazer upload de múltiplos arquivos por aula e ver a % de completude do curso,
**so that** eu gerencie o conteúdo de forma mais eficiente.

---

## Acceptance Criteria

1. No formulário de criação/edição de aula, o campo de upload aceita **múltiplos arquivos** simultaneamente (drag-and-drop ou file picker multi-select)
2. Upload aceita **qualquer tipo de arquivo** (não apenas PDF) — inclui imagens, planilhas, documentos, vídeos, etc.
3. Durante o upload de múltiplos arquivos, uma barra de progresso individual é exibida para cada arquivo
4. Os arquivos são salvos no Supabase Storage e referenciados na aula (tabela `lesson_files` ou array em `lessons`)
5. Na listagem de aulas, cada aula mostra a quantidade de arquivos anexados
6. O admin pode remover arquivos individuais de uma aula
7. No painel admin, na página de edição de produto, exibir **barra de progresso de completude do curso**
8. A barra de completude calcula: % de aulas que possuem vídeo configurado + descrição preenchida + status publicado
9. A barra mostra breakdown: "X/Y aulas com vídeo", "X/Y aulas com descrição", "X/Y aulas publicadas"
10. A barra de completude é visível também na listagem de produtos como coluna/indicador

## Dev Notes

### Schema: lesson_files

```sql
CREATE TABLE lesson_files (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    lesson_id UUID NOT NULL REFERENCES lessons(id) ON DELETE CASCADE,
    file_name TEXT NOT NULL,
    file_url TEXT NOT NULL,
    file_type TEXT NOT NULL DEFAULT '',
    file_size BIGINT NOT NULL DEFAULT 0,
    sort_order INTEGER NOT NULL DEFAULT 0,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_lesson_files_lesson ON lesson_files(lesson_id);
```

### Cálculo de Completude

```
completude_geral = (aulas_com_video + aulas_com_descricao + aulas_publicadas) / (total_aulas * 3) * 100
```

### Migração do campo pdf_url

- O campo `pdf_url` existente em `lessons` pode ser mantido para retrocompatibilidade
- Novos uploads usam a tabela `lesson_files`
- Considerar migração futura para mover PDFs existentes para `lesson_files`

### Considerações

- Limite de tamanho por arquivo: 50MB (Supabase Storage)
- Limite de arquivos por aula: 20 (soft limit na UI)
- Tipos MIME: aceitar todos, exibir ícone genérico para tipos não reconhecidos

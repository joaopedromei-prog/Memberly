## Story

**As an** admin,
**I want** importar membros via CSV, exportar lista de membros, e conceder/revogar acesso em lote,
**so that** eu consiga gerenciar membros em escala sem fazer um por um.

---

## Context

### Estado Atual

A página de membros (`/admin/members` ou equivalente) lista membros individualmente. Não existe funcionalidade de import/export nem ações em lote. A gestão de acesso é feita membro a membro.

### Mudança Necessária

Adicionar 3 capacidades: (1) importar membros via CSV com auto-criação de profiles e acesso, (2) exportar lista de membros como CSV, (3) seleção múltipla com ações em lote (conceder/revogar acesso a um produto).

### Decisões Técnicas

- Import CSV usa `supabaseAdmin` (service role) para criar profiles via `auth.admin.createUser()` — bypassa auth flow normal
- Export CSV é gerado server-side e retornado como download
- Batch access usa transação (ou loop com tratamento individual) para grant/revoke em massa
- File parsing do CSV acontece client-side (FileReader + parsing manual ou lib leve) para preview antes do envio

---

## Acceptance Criteria

1. Botão "Importar CSV" na página de membros abre modal com área de drop/upload para arquivo CSV
2. CSV aceita colunas: `email`, `full_name`, `product_slug` (uma linha por membro-produto, encoding UTF-8). Preview dos dados antes de confirmar. Headers case-insensitive, campos com vírgulas devem estar entre aspas
3. Import cria profiles (se não existem) via `auth.admin.createUser()` com senha aleatória (membro recebe email de redefinição via Supabase Auth) e concede acesso ao produto. Resumo final: X criados, Y já existentes, Z erros (com detalhes)
4. Botão "Exportar CSV" gera download com colunas: email, full_name, created_at, produtos com acesso (comma-separated)
5. Checkboxes na lista de membros para seleção individual + "selecionar todos da página"
6. Barra de ações flutuante quando há seleção: "{N} selecionados | Conceder Acesso | Revogar Acesso"
7. "Conceder Acesso" abre modal para selecionar produto → confirma → aplica para todos selecionados
8. "Revogar Acesso" abre modal para selecionar produto → confirmação com texto de alerta → aplica
9. Build, typecheck e lint passam

## Dev Notes

### CSV Format (Import)

```csv
email,full_name,product_slug
joao@email.com,João Silva,curso-calistenia
maria@email.com,Maria Santos,curso-calistenia
```

### CSV Parsing Rules

- Encoding: **UTF-8** (rejeitar arquivos com BOM inválido)
- Headers: case-insensitive (`Email` = `email` = `EMAIL`)
- Campos com vírgulas: devem estar entre aspas (`"Silva, João"`)
- Linhas vazias: ignorar silenciosamente
- Email inválido: registrar como erro na linha

### Import Flow

```
1. Admin faz upload do CSV
2. Client-side: parse CSV (UTF-8, aspas) → preview em tabela
3. Admin confirma → POST /api/members/import
4. Server: para cada linha:
   a. Busca profile por email → se não existe:
      - auth.admin.createUser({email, password: randomUUID(), email_confirm: true})
      - insert profile (full_name, role: 'member')
      - auth.admin.generateLink({type: 'recovery', email}) → membro recebe email pra criar senha
   b. Se profile já existe: skip criação, usar profile_id existente
   c. Busca product por slug → se não existe, registra erro na linha
   d. Insert member_access (on conflict do nothing)
5. Retorna resumo: {created, existing, errors: [{row, email, reason}]}
```

### Batch Access API

```typescript
// POST /api/members/batch-access
{
  profile_ids: ["uuid1", "uuid2", "uuid3"],
  product_id: "uuid-product",
  action: "grant" // ou "revoke"
}
```

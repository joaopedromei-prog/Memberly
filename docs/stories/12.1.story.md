## Story

**Como** administrador da plataforma,
**Quero** que todos os membros criados (via admin ou webhook) recebam a senha padrão `734650`,
**Para que** eu possa comunicar uma senha simples e unificada a todos os alunos, sem depender de emails de convite ou reset.

## Acceptance Criteria

1. **AC1** — Ao criar um membro via painel admin (single add), o usuário Supabase Auth deve ser criado com a senha `734650` (pré-confirmado, sem email de verificação).
2. **AC2** — Ao importar membros via CSV (bulk import), cada usuário Supabase Auth deve ser criado com a senha `734650` (pré-confirmado, sem email de verificação).
3. **AC3** — Ao provisionar membro via webhook (Payt), o usuário Supabase Auth deve ser criado com `createUser()` usando a senha `734650` em vez de `inviteUserByEmail()`.
4. **AC4** — Nenhum email de reset/convite deve ser enviado automaticamente na criação do membro (removido dos 3 fluxos).
5. **AC5** — A senha padrão deve ser definida como variável de ambiente `DEFAULT_MEMBER_PASSWORD` em `.env.local`, com fallback para `734650`. Nunca hardcoded diretamente nos endpoints.
6. **AC6** — O fluxo "Esqueci minha senha" (`/forgot-password`) deve continuar funcionando normalmente como único meio de alterar a senha.
7. **AC7** — Membros existentes não são afetados — a mudança aplica-se apenas a novos cadastros.

### Contexto Técnico

- Supabase Auth `admin.createUser()` aceita `password` + `email_confirm: true` para criar usuário pré-confirmado
- A senha `734650` atende o mínimo de 6 caracteres do Supabase Auth
- O método `inviteUserByEmail()` no webhook será substituído por `createUser()` com a mesma lógica de busca (`listUsers` → se não existe → cria)

### Edge Cases

- **Email duplicado no Auth:** `createUser()` retorna erro se o email já existe. Tratar com try/catch e retornar o user existente (mesma lógica de `findOrCreateMember`)
- **Email existe no Auth mas não no profiles:** Criar registro no profiles normalmente — o trigger `handle_new_user()` pode não ter rodado. Fazer upsert no profiles como fallback.
- **Senha via env var:** Se `DEFAULT_MEMBER_PASSWORD` não estiver definida em `.env.local`, usar fallback `'734650'`. Isso garante que o sistema funciona mesmo sem a variável configurada.

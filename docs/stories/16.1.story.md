## Story

**Como** sistema de gamificacao,
**Quero** tabelas `badges`, `member_badges` e `streaks` no Supabase com RLS e indexes adequados,
**Para que** o sistema possa armazenar badges disponiveis, badges desbloqueados por membros e streaks de atividade diaria.

## Acceptance Criteria

1. **AC1** — Tabela `badges` criada com colunas: `id` (UUID PK, default `uuid_generate_v4()`), `name` (text NOT NULL), `description` (text NOT NULL default ''), `icon_url` (text, nullable), `criteria` (jsonb NOT NULL), `active` (boolean NOT NULL default true), `created_at` (timestamptz NOT NULL default NOW()), `updated_at` (timestamptz NOT NULL default NOW()).
2. **AC2** — Tabela `member_badges` criada com colunas: `id` (UUID PK, default `uuid_generate_v4()`), `profile_id` (UUID FK -> profiles, NOT NULL), `badge_id` (UUID FK -> badges, NOT NULL), `unlocked_at` (timestamptz NOT NULL default NOW()), com constraint UNIQUE em `(profile_id, badge_id)`.
3. **AC3** — Tabela `streaks` criada com colunas: `id` (UUID PK, default `uuid_generate_v4()`), `profile_id` (UUID FK -> profiles, NOT NULL, UNIQUE), `current_streak` (integer NOT NULL default 0), `longest_streak` (integer NOT NULL default 0), `last_activity_date` (date, nullable), `created_at` (timestamptz NOT NULL default NOW()), `updated_at` (timestamptz NOT NULL default NOW()).
4. **AC4** — Foreign keys em `member_badges` e `streaks` com `ON DELETE CASCADE` para `profiles`. FK `badge_id` em `member_badges` com `ON DELETE CASCADE` para `badges`.
5. **AC5** — RLS habilitado nas 3 tabelas: membros podem ler todos os badges (SELECT); membros veem apenas seus proprios `member_badges` e `streaks`; admins tem acesso total (ALL) em todas as tabelas.
6. **AC6** — Index `idx_member_badges_profile` em `member_badges(profile_id)`.
7. **AC7** — Index `idx_member_badges_badge` em `member_badges(badge_id)`.
8. **AC8** — Index `idx_streaks_profile` em `streaks(profile_id)`.
9. **AC9** — Type definitions atualizadas em `src/types/database.ts` com tipos `badges`, `member_badges` e `streaks` (Row, Insert, Update) e convenience aliases.
10. **AC10** — Migration nomeada `20260320000004_create_gamification.sql`.

## Dev Notes

### DDL Reference

```sql
-- Badges table
CREATE TABLE badges (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    name TEXT NOT NULL,
    description TEXT NOT NULL DEFAULT '',
    icon_url TEXT,
    criteria JSONB NOT NULL,
    active BOOLEAN NOT NULL DEFAULT true,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Member badges (unlocked badges per member)
CREATE TABLE member_badges (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    profile_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
    badge_id UUID NOT NULL REFERENCES badges(id) ON DELETE CASCADE,
    unlocked_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE(profile_id, badge_id)
);

CREATE INDEX idx_member_badges_profile ON member_badges(profile_id);
CREATE INDEX idx_member_badges_badge ON member_badges(badge_id);

-- Streaks table (one row per member)
CREATE TABLE streaks (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    profile_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE UNIQUE,
    current_streak INTEGER NOT NULL DEFAULT 0,
    longest_streak INTEGER NOT NULL DEFAULT 0,
    last_activity_date DATE,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_streaks_profile ON streaks(profile_id);
```

### RLS Pattern (seguir migration 20260320000002_create_notifications.sql)

```sql
-- Badges RLS (all members can read, only admins can write)
ALTER TABLE badges ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Anyone can view active badges"
    ON badges FOR SELECT
    USING (true);

CREATE POLICY "Admins have full access to badges"
    ON badges FOR ALL
    USING ((auth.jwt() -> 'user_metadata' ->> 'role') = 'admin');

-- Member badges RLS
ALTER TABLE member_badges ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Members can view own badges"
    ON member_badges FOR SELECT
    USING (auth.uid() = profile_id);

CREATE POLICY "Admins have full access to member_badges"
    ON member_badges FOR ALL
    USING ((auth.jwt() -> 'user_metadata' ->> 'role') = 'admin');

-- Service role bypass: INSERT will be done via admin client (service_role)

-- Streaks RLS
ALTER TABLE streaks ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Members can view own streak"
    ON streaks FOR SELECT
    USING (auth.uid() = profile_id);

CREATE POLICY "Admins have full access to streaks"
    ON streaks FOR ALL
    USING ((auth.jwt() -> 'user_metadata' ->> 'role') = 'admin');
```

### Edge Cases

- **UNIQUE constraint em `streaks(profile_id)`:** Garante um unico registro de streak por membro. Usar UPSERT ao atualizar.
- **UNIQUE constraint em `member_badges(profile_id, badge_id)`:** Previne duplicatas de badges desbloqueados.
- **Cascade delete:** Se um badge for deletado, `member_badges` associados sao removidos automaticamente.
- **Criteria jsonb:** Flexivel para adicionar novos tipos de badge sem migration.

## Story

**As a** membro,
**I want** ver meus produtos em uma vitrine visual estilo Netflix,
**so that** eu encontre facilmente o conteúdo que quero assistir.

---

## Acceptance Criteria

1. Página home (`/`) exibe hero banner rotativo com o produto mais recente ou featured
2. Abaixo do hero, carrosséis horizontais ("Continue Assistindo" se houver progresso, "Meus Cursos" com produtos do membro)
3. Cada card de produto exibe: banner, título, barra de progresso (% de aulas concluídas)
4. Cards são clicáveis e levam à página do produto
5. Se o membro não tem produtos, exibir mensagem amigável ("Você ainda não tem cursos. Entre em contato com o suporte.")
6. Carrosséis com scroll horizontal (drag em desktop, swipe em mobile)
7. Layout responsivo: 4-5 cards em desktop, 2 cards em tablet, 1.5 cards em mobile (mostrando parcialmente o próximo)
8. Loading skeletons enquanto dados carregam

### Supabase Query Patterns [Source: architecture.md#8-database-schema, #9-frontend-architecture]

**Fetch produtos do membro com progresso (RSC — server-side):**
```typescript
const supabase = await createServerSupabaseClient();
const { data: { user } } = await supabase.auth.getUser();

// Produtos com acesso
const { data: products } = await supabase
  .from('products')
  .select(`
    id, title, description, banner_url, slug, sort_order,
    modules (
      id,
      lessons ( id )
    )
  `)
  .eq('is_published', true)
  .order('sort_order');

// Progresso do membro
const { data: progress } = await supabase
  .from('lesson_progress')
  .select('lesson_id, completed')
  .eq('profile_id', user.id)
  .eq('completed', true);

// RLS filtra automaticamente via member_access
```

### Animation Tokens [Source: front-end-spec.md#9-animation-micro-interactions]

| Animation | Duration | Easing | Property |
|-----------|----------|--------|----------|
| Card hover | 150ms | ease-out | transform: scale(1.05), box-shadow |
| Carousel scroll | 300ms | ease-in-out | transform: translateX |
| Progress bar update | 500ms | ease-in-out | width + color |
| Page transition | 200ms | ease-out | opacity |
| Skeleton shimmer | 1.5s | linear (loop) | background-position |

**IMPORTANT:** Respect `prefers-reduced-motion: reduce` — disable all animations except opacity fades.

### Accessibility [Source: front-end-spec.md#7-accessibility-requirements]

- Alt text em todas as imagens de banner do produto
- `aria-label` nos botões de navegação do carrossel
- `role="region"` + `aria-label` nas seções de carrossel
- Focus indicators visíveis (2px solid ring) em cards e botões
- Navegação por teclado: Tab entre cards, Enter para navegar
- Touch targets >= 44x44px em mobile
- Headings hierárquicos: h1 (hero), h2 (seções), no h3

### Performance [Source: front-end-spec.md#10-performance-considerations]

- RSC para data fetching (zero JS para conteúdo estático)
- Imagens de banner: `next/image` com WebP, `srcset`, lazy loading abaixo do fold
- Hero banner: `priority` loading (above the fold)
- Skeleton screens via `loading.tsx` (não spinners)
- Aspect ratio definido em containers de imagem (previne CLS)
- Target LCP < 2.5s, CLS < 0.1

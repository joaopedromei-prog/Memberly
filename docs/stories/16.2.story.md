## Story

**Como** membro da plataforma,
**Quero** que badges sejam automaticamente desbloqueados quando eu atingir os criterios necessarios,
**Para que** eu receba reconhecimento imediato pelo meu progresso e engajamento.

## Acceptance Criteria

1. **AC1** — Servico `src/lib/gamification/badge-engine.ts` exporta funcao `evaluateBadges(profileId: string): Promise<string[]>` que retorna array de IDs dos badges recem-desbloqueados.
2. **AC2** — O servico busca todos os badges ativos (`active = true`) e filtra os que o membro ainda nao possui (sem registro em `member_badges`).
3. **AC3** — Criterio `FIRST_LESSON`: Verifica se o membro tem pelo menos 1 registro em `lesson_progress` com `completed = true`.
4. **AC4** — Criterio `COURSE_COMPLETE`: Verifica se o membro completou 100% das aulas publicadas de pelo menos 1 produto (reutilizar logica de completion check).
5. **AC5** — Criterio `STREAK_7`: Verifica se `streaks.current_streak >= 7` para o membro.
6. **AC6** — Criterio `STREAK_30`: Verifica se `streaks.current_streak >= 30` para o membro.
7. **AC7** — Criterio `COMMENTS_10`: Verifica se o membro tem pelo menos 10 registros em `comments`.
8. **AC8** — Criterio `EXPLORER_3`: Verifica se o membro tem `member_access` a pelo menos 3 produtos distintos.
9. **AC9** — Criterio `LESSONS_50`: Verifica se o membro tem pelo menos 50 registros em `lesson_progress` com `completed = true`.
10. **AC10** — Para cada badge que atende ao criterio e o membro ainda nao possui, o servico insere em `member_badges` usando admin client (service role).
11. **AC11** — Interface `BadgeCriteria` e mapa de criterios exportados para extensibilidade.
12. **AC12** — Testes unitarios para cada criterio e para a funcao principal `evaluateBadges`.

### Badge Engine Pattern

```typescript
// src/lib/gamification/badge-engine.ts
import { createAdminClient } from '@/lib/supabase/admin';
import { criteriaCheckers } from './criteria';

export async function evaluateBadges(profileId: string): Promise<string[]> {
  const adminClient = createAdminClient();

  // 1. Get all active badges
  const { data: badges } = await adminClient
    .from('badges')
    .select('id, criteria')
    .eq('active', true);

  if (!badges?.length) return [];

  // 2. Get member's existing badges
  const { data: existing } = await adminClient
    .from('member_badges')
    .select('badge_id')
    .eq('profile_id', profileId);

  const existingIds = new Set(existing?.map((b) => b.badge_id) ?? []);

  // 3. Filter badges member doesn't have
  const candidates = badges.filter((b) => !existingIds.has(b.id));
  if (!candidates.length) return [];

  // 4. Evaluate each candidate
  const unlocked: string[] = [];

  for (const badge of candidates) {
    const criteria = badge.criteria as { type: string; threshold: number };
    const checker = criteriaCheckers[criteria.type];
    if (!checker) continue;

    const met = await checker(profileId, criteria.threshold, adminClient);
    if (met) {
      await adminClient.from('member_badges').insert({
        profile_id: profileId,
        badge_id: badge.id,
      });
      unlocked.push(badge.id);
    }
  }

  return unlocked;
}
```

### Criteria Checker Pattern

```typescript
// src/lib/gamification/criteria/first-lesson.ts
import type { SupabaseClient } from '@supabase/supabase-js';

export async function checkFirstLesson(
  profileId: string,
  _threshold: number,
  adminClient: SupabaseClient
): Promise<boolean> {
  const { count } = await adminClient
    .from('lesson_progress')
    .select('id', { count: 'exact', head: true })
    .eq('profile_id', profileId)
    .eq('completed', true);

  return (count ?? 0) >= 1;
}
```

### Edge Cases

- **Race condition:** Se `evaluateBadges` for chamado concorrentemente, o UNIQUE constraint em `(profile_id, badge_id)` previne duplicatas. Tratar `unique_violation` silenciosamente.
- **Performance:** Criterios sao avaliados sequencialmente para evitar sobrecarga. Para v1 com ~7 badges, performance e aceitavel.
- **COURSE_COMPLETE:** Precisa verificar todos os produtos do membro, nao apenas um. Usar `member_access` para listar produtos e verificar completion de cada um.
- **STREAK checks:** Ambos `STREAK_7` e `STREAK_30` usam a mesma funcao `checkStreak` mas com threshold diferente via criteria jsonb.

### Testing

- **Cada criterio:** Mock admin client, testar retorno `true`/`false` com diferentes estados
- **evaluateBadges:** Mock badges, existing member_badges, criterios. Verificar que apenas badges novos sao desbloqueados
- **UNIQUE violation:** Verificar que duplicate insert e tratado graciosamente
- **Sem badges ativos:** Retorna array vazio

## Dev Notes

### Context

A auditoria de design system (2026-03-13) identificou cores usadas frequentemente que NÃO possuem tokens:

| Cor | Uso | Ocorrências |
|-----|-----|-------------|
| `#B3B3B3` | Texto secundário (member) | 15+ |
| `#46D369` | Success/completion (member) | 15+ |
| `#F59E0B` | Warning/lock (ambos) | 4+ |
| `#2563EB` | Primary blue (admin) | 35+ |
| `#1D4ED8` | Blue hover (admin) | 3+ |
| `#0A0A0A` | Darkest background (auth) | 15+ |
| `#1F1F1F` | Input background (member) | 8+ |

Tipografia custom inline: `text-[28px]`, `text-[40px]`, `text-[48px]` em `ProductHero.tsx`.

### Decisões Técnicas

- Novos tokens seguem o padrão existente: `--color-{semantic-name}`
- Tokens de tipografia seguem: `--font-size-{semantic-name}`
- Cores admin (blue) ficam como tokens separados dos member (dark)
- Manter ambos os temas funcionais (light admin / dark member)
- **Separação de escopo com Story 11.1:** Esta story trata APENAS de hex que NÃO tinham token existente. A Story 11.1 trata dos que já tinham.

### Tokens Propostos para globals.css

```css
@theme {
  /* Existing tokens... */

  /* New color tokens */
  --color-text-muted: #B3B3B3;
  --color-accent-success: #46D369;
  --color-accent-warning: #F59E0B;
  --color-admin-primary: #2563EB;
  --color-admin-primary-hover: #1D4ED8;
  --color-dark-bg-deep: #0A0A0A;
  --color-dark-input: #1F1F1F;

  /* Typography tokens */
  --font-size-display: 48px;
  --font-size-hero: 40px;
  --font-size-subtitle: 28px;
}
```

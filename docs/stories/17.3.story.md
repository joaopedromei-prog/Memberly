## Story

**Como** cliente que acabou de comprar um produto na Payt,
**Quero** receber um email de boas-vindas com um link para acessar a plataforma,
**Para que** eu saiba que minha compra foi processada e consiga acessar o conteudo imediatamente.

## Acceptance Criteria

1. **AC1** — Apos o webhook da Payt conceder acesso com sucesso (insert em `member_access`), o sistema envia email de boas-vindas automaticamente.
2. **AC2** — O email usa o template configurado em `site_settings` (subject e body com variaveis substituidas).
3. **AC3** — O `{{login_url}}` no template aponta para a pagina de login da plataforma (`NEXT_PUBLIC_APP_URL/login`).
4. **AC4** — Se o envio do email falhar, o erro e logado no `webhook_logs` (campo `error_message` atualizado) mas o acesso ao produto NAO e revogado — o acesso ja foi concedido.
5. **AC5** — Se `welcome_email_active` estiver `false`, o email nao e enviado (respeitando configuracao do admin).
6. **AC6** — O webhook handler retorna 200 independente do resultado do envio de email (email e best-effort, nao bloqueia resposta).
7. **AC7** — Log de auditoria: campo `email_sent` (boolean) e `email_error` (string nullable) adicionados ao registro de `webhook_logs` para rastreabilidade.
8. **AC8** — Testes de integracao atualizados para cobrir: envio ok, envio falho, template inativo.

## Dev Notes

### Fluxo atualizado do webhook

```
Webhook POST -> Validar -> Logar -> Verificar pagamento
  -> Lookup produto -> Provisionar membro -> Conceder acesso
  -> [NOVO] Enviar email de boas-vindas (best-effort)
  -> Retornar 200
```

### Dependencias

- **Story 17.1** — `sendWelcomeEmail` deve estar implementada
- **Story 17.2** — Login via magic link deve estar funcional (o email direciona para `/login`)

### Email e best-effort

O envio de email NAO deve bloquear a resposta do webhook. A Payt espera resposta rapida (< 5s). O email pode ser enviado de forma sincrona mas com timeout curto, ou o erro simplesmente e logado.

### Migration

```sql
ALTER TABLE webhook_logs
  ADD COLUMN email_sent BOOLEAN NOT NULL DEFAULT false,
  ADD COLUMN email_error TEXT;
```

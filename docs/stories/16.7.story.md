### API Routes Pattern (seguir products)

```typescript
// src/app/api/admin/badges/route.ts
import { requireAdmin } from '@/lib/utils/auth-guard';
import { apiSuccess, apiError } from '@/lib/utils/api-response';
import { createServerClient } from '@/lib/supabase/server';

export async function GET() {
  const admin = await requireAdmin();
  if (!admin) return apiError('Forbidden', 403);

  const supabase = await createServerClient();
  const { data, error } = await supabase
    .from('badges')
    .select('*')
    .order('created_at', { ascending: false });

  if (error) return apiError(error.message, 500);
  return apiSuccess(data);
}

export async function POST(request: Request) {
  const admin = await requireAdmin();
  if (!admin) return apiError('Forbidden', 403);

  const body = await request.json();
  const supabase = await createServerClient();

  const { data, error } = await supabase
    .from('badges')
    .insert({
      name: body.name,
      description: body.description ?? '',
      icon_url: body.icon_url ?? null,
      criteria: { type: body.criteria_type, threshold: body.threshold ?? 1 },
      active: body.active ?? true,
    })
    .select()
    .single();

  if (error) return apiError(error.message, 500);
  return apiSuccess(data, 201);
}
```

### Edge Cases

- **Badge em uso:** Ao deletar badge que membros ja desbloquearam, CASCADE remove `member_badges`. Exibir contagem de membros afetados na confirmacao.
- **Criterio duplicado:** Multiplos badges podem ter o mesmo tipo de criterio com thresholds diferentes (ex: LESSONS_50 e LESSONS_100 futuro).
- **Icon URL:** Aceitar URL externa ou path relativo. Validar que e URL valida.
- **Threshold:** Obrigatorio para todos os criterios, default 1 para FIRST_LESSON e COURSE_COMPLETE.

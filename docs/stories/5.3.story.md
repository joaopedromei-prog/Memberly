## Story

**As a** admin,
**I want** que a IA gere imagens de banner para módulos e produtos,
**so that** a área de membros tenha visual completo sem necessidade de designer.

---

## Acceptance Criteria

1. No wizard de geração, opção "Gerar banners automaticamente" (checkbox, default: off)
2. Se ativado: para cada módulo e para o produto, enviar descrição de banner para Gemini API (Google) com capacidade nativa de geração de imagens
3. Banners gerados exibidos no preview para aprovação
4. Admin pode aceitar, rejeitar (usar placeholder) ou regenerar cada banner individualmente
5. Banners aprovados são salvos no Supabase Storage e vinculados aos respectivos módulos/produto
6. Se API de imagem indisponível ou feature desabilitada: fallback para banners placeholder com gradient + texto

## Dev Notes

### Gemini API Integration [Source: architecture.md#6-external-apis]

```typescript
// src/lib/ai/gemini-client.ts
import { GoogleGenerativeAI } from '@google/generative-ai';

const genAI = new GoogleGenerativeAI(process.env.GEMINI_API_KEY!);

export async function generateBanner(description: string): Promise<string> {
  const model = genAI.getGenerativeModel({ model: 'gemini-2.0-flash-exp' });

  const result = await model.generateContent({
    contents: [
      {
        role: 'user',
        parts: [
          {
            text: buildBannerPrompt(description),
          },
        ],
      },
    ],
    generationConfig: {
      responseModalities: ['image', 'text'],
    },
  });

  const response = result.response;
  const imagePart = response.candidates?.[0]?.content?.parts?.find(
    (part) => part.inlineData?.mimeType?.startsWith('image/')
  );

  if (!imagePart?.inlineData?.data) {
    throw new Error('No image generated by Gemini API');
  }

  return imagePart.inlineData.data; // base64 string
}
```

```typescript
export function buildBannerPrompt(description: string): string {
  return `Generate a professional banner image for an online course platform.

Requirements:
- Aspect ratio: 16:9 (1920x1080 or similar)
- Style: Modern, clean, professional
- No text overlay (text will be added by the platform)
- Color palette: Rich, vibrant colors appropriate to the topic
- Feel: Premium, educational, inviting

Banner description: ${description}

Generate a high-quality banner image that matches this description.`;
}
```

### Base64 to Supabase Storage Pipeline

```typescript
// src/lib/storage/banner-upload.ts
import { createAdminClient } from '@/lib/supabase/admin';

export async function uploadBannerFromBase64(
  base64Data: string,
  path: string, // e.g., 'banners/curso-marketing/product.png'
  mimeType: string = 'image/png'
): Promise<string> {
  const adminClient = createAdminClient();
  const buffer = Buffer.from(base64Data, 'base64');

  const { data, error } = await adminClient.storage
    .from('banners')
    .upload(path, buffer, {
      contentType: mimeType,
      upsert: true,
    });

  if (error) throw new Error(`Storage upload failed: ${error.message}`);

  const { data: urlData } = adminClient.storage
    .from('banners')
    .getPublicUrl(data.path);

  return urlData.publicUrl;
}
```

### Rate Limit Considerations

- Gemini free tier: 15 RPM
- For a product with 10 modules: 11 banner generations (1 product + 10 modules)
- Sequential with 100ms delay between calls to avoid rate limits
- If rate limited (429): exponential backoff, skip and use placeholder

### API Spec [Source: architecture.md#5-api-specification]

| Method | Route | Auth | Description |
|--------|-------|------|-------------|
| POST | `/api/ai/generate-banner` | Admin | Gerar banner via Gemini API |

### Supabase Storage Bucket Configuration

O bucket `banners` deve ser criado no Supabase Dashboard (ou via migration) com:
- Public access enabled (banners são imagens públicas)
- File size limit: 5MB
- Allowed MIME types: image/png, image/jpeg, image/webp

### Coding Standards [Source: architecture.md#16-coding-standards]

- **Absolute Imports:** Always use `@/` prefix
- **Environment Variables:** `GEMINI_API_KEY` server-side only (no NEXT_PUBLIC_ prefix)
- **Error Handling:** Graceful fallback to placeholder — never crash the wizard on image generation failure
- **Storage:** Use admin client for upload (webhook/admin context), public URL for display

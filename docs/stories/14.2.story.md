## Story

**Como** membro que completou 100% das aulas de um produto,
**Quero** que o sistema gere um certificado de conclusão em PDF,
**Para que** eu possa baixar e compartilhar meu certificado com dados do meu nome, produto e data de conclusão.

## Acceptance Criteria

1. **AC1** — Endpoint `POST /api/certificates/generate` aceita `{ productId: string }` e retorna `{ certificate: { id, hash, certificate_url, issued_at } }` com status 201.
2. **AC2** — Antes de gerar, o endpoint verifica se o membro completou 100% das aulas publicadas do produto via `lesson_progress`. Se incompleto, retorna erro `INCOMPLETE_PRODUCT` (400).
3. **AC3** — Se já existe certificado para o par `(profile_id, product_id)` na tabela `certificates`, retorna o existente com status 200 (idempotência).
4. **AC4** — O PDF é gerado com: nome do membro (de `profiles.full_name`), título do produto (de `products.title`), data de emissão, e hash de validação.
5. **AC5** — O PDF é armazenado no Supabase Storage bucket `certificates/` com path `{profile_id}/{product_id}.pdf`.
6. **AC6** — O hash é gerado como SHA-256 de `{profile_id}:{product_id}:{timestamp}` e é único.
7. **AC7** — Um registro é criado na tabela `certificates` com `profile_id`, `product_id`, `certificate_url`, `hash`, `issued_at`.
8. **AC8** — O endpoint requer autenticação (membro logado). Apenas o próprio membro pode gerar seu certificado.
9. **AC9** — O endpoint verifica que o membro tem `member_access` ao produto antes de gerar.
10. **AC10** — O PDF tem formato A4 landscape, tema dark (Netflix-style), com logo/branding do site.

### Completion Check Logic

Query todas as `lessons` publicadas (`is_published = true`) do produto via `modules → lessons`, depois compara com `lesson_progress` do membro:

```typescript
// 1. Get all published lessons for the product
const { data: modules } = await supabase
  .from('modules')
  .select('lessons(id)')
  .eq('product_id', productId)
  .eq('lessons.is_published', true);

const allLessonIds = modules?.flatMap(m => m.lessons.map(l => l.id)) ?? [];

// 2. Get completed lessons
const { data: progress } = await supabase
  .from('lesson_progress')
  .select('lesson_id')
  .eq('profile_id', profileId)
  .eq('completed', true)
  .in('lesson_id', allLessonIds);

const completedCount = progress?.length ?? 0;
const isComplete = completedCount === allLessonIds.length && allLessonIds.length > 0;
```

### PDF Generation Pattern

Use `@react-pdf/renderer` para server-side rendering do PDF. O template deve ser Netflix-style:
- Background: `#141414` (dark-bg)
- Accent: `#e50914` (Netflix red)
- Font: sans-serif (Helvetica built-in no react-pdf)
- Layout: A4 landscape
- Conteúdo: nome do membro, nome do produto, data formatada, hash de validação

### Storage Upload Pattern (seguir `banner-upload.ts`)

```typescript
import { createAdminClient } from '@/lib/supabase/admin';

const adminClient = createAdminClient();
const { data, error } = await adminClient.storage
  .from('certificates')
  .upload(`${profileId}/${productId}.pdf`, buffer, {
    contentType: 'application/pdf',
    upsert: true,
  });
```

### Hash Generation

```typescript
import { createHash } from 'crypto';

export function generateCertificateHash(profileId: string, productId: string): string {
  const timestamp = Date.now().toString();
  return createHash('sha256')
    .update(`${profileId}:${productId}:${timestamp}`)
    .digest('hex');
}
```

### Edge Cases

- **Produto sem aulas:** Se o produto não tem aulas publicadas (`allLessonIds.length === 0`), retornar erro `NO_LESSONS` (400) — não é possível "completar" um produto vazio.
- **Certificado já existe:** Retornar o existente (idempotente), não gerar novo PDF.
- **Storage bucket não existe:** Criar via migration ou documentar setup manual.
- **PDF generation timeout:** O react-pdf roda server-side e pode ser lento. Considerar timeout de 30s.

3. Set as **public** bucket (for read access to certificate PDFs)

## Story

**As a** membro,
**I want** ver uma interface visual estilo Netflix ao fazer login,
**so that** minha experiência de navegação seja familiar e agradável.

---

## Acceptance Criteria

1. Layout de membro (`/`) com header minimal: logo placeholder, nome do membro, botão de logout
2. Página home com layout Netflix-like: hero banner grande no topo (placeholder), área de carrosséis horizontais (placeholder)
3. Footer simples com links de suporte e copyright
4. Tema escuro (Netflix-inspired) com paleta neutra: fundo escuro (#141414), texto branco, acentos em cor primária
5. Layout totalmente responsivo (mobile-first): cards se ajustam, hero redimensiona, navegação simplifica
6. Fontes >= 16px body, botões >= 44px touch target
7. Transições suaves de navegação entre páginas

### Netflix-like Dark Theme Design Tokens

```typescript
// tailwind.config.ts — extend colors
const config = {
  theme: {
    extend: {
      colors: {
        'dark-bg': '#141414',
        'dark-surface': '#1a1a1a',
        'dark-card': '#2a2a2a',
        'dark-border': '#333333',
        primary: {
          DEFAULT: '#e50914',
          hover: '#f40612',
        },
      },
    },
  },
};
```

**Color usage:**
| Element | Color | Tailwind Class |
|---------|-------|---------------|
| Page background | #141414 | `bg-dark-bg` or `bg-[#141414]` |
| Card background | #2a2a2a | `bg-dark-card` |
| Surface (elevated) | #1a1a1a | `bg-dark-surface` |
| Border | #333333 | `border-dark-border` |
| Primary text | white | `text-white` |
| Secondary text | #a3a3a3 | `text-neutral-400` |
| Primary accent | #e50914 | `bg-primary text-white` |
| Footer background | #0a0a0a | `bg-[#0a0a0a]` |

### Member Layout Pattern

```typescript
// src/app/(member)/layout.tsx
import { MemberHeader } from '@/components/member/MemberHeader';
import { MemberFooter } from '@/components/member/MemberFooter';

export default function MemberLayout({ children }: { children: React.ReactNode }) {
  return (
    <div className="min-h-screen bg-[#141414] text-white">
      <MemberHeader />
      <main>{children}</main>
      <MemberFooter />
    </div>
  );
}
```

### Carousel Scroll Pattern

```typescript
// Horizontal scroll with snap points
<div className="overflow-x-auto flex gap-3 snap-x snap-mandatory scrollbar-hide pb-4">
  {items.map((item) => (
    <div key={item.id} className="snap-start flex-shrink-0 w-[250px] md:w-[300px] lg:w-[350px]">
      <ProductCard {...item} />
    </div>
  ))}
</div>
```

**Hide scrollbar CSS** (add to global styles or Tailwind plugin):
```css
.scrollbar-hide {
  -ms-overflow-style: none;
  scrollbar-width: none;
}
.scrollbar-hide::-webkit-scrollbar {
  display: none;
}
```

### Accessibility Requirements (AC: 6)

| Element | Requirement | Implementation |
|---------|-------------|----------------|
| Body font | >= 16px | Tailwind `text-base` (default 16px) |
| Button touch target | >= 44px | `min-h-[44px] min-w-[44px]` |
| Color contrast | WCAG AA | White text on #141414 = 15.4:1 ratio (passes AAA) |
| Focus indicators | Visible | `focus:ring-2 focus:ring-primary focus:outline-none` |

### Responsive Breakpoints

| Breakpoint | Hero Height | Card Width | Header Padding | Cards Visible |
|-----------|-------------|------------|----------------|---------------|
| Mobile (< 768px) | 40vh | 250px | px-4 | ~1.5 |
| Tablet (768px) | 48vh | 300px | px-8 | ~2.5 |
| Desktop (1024px+) | 56vh | 350px | px-16 | ~4-5 |

### Coding Standards [Source: architecture.md#16-coding-standards]

- **Components:** PascalCase in `src/components/member/` directory
- **Absolute Imports:** `@/components/member/HeroBanner`, `@/stores/auth-store`
- **Server vs Client:** Layout and page can be Server Components; Header needs `'use client'` for auth store + logout; Footer can be Server Component
- **Styling:** Tailwind CSS with custom dark theme tokens, `cn()` for conditional classes
- **Mobile-first:** Start with mobile styles, add `md:` and `lg:` breakpoints for larger screens

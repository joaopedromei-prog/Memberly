## Dev Notes

### Context

Gradientes estão definidos inline em arrays dentro de componentes específicos:

1. **`MemberList.tsx`** — `AVATAR_GRADIENTS` (8 gradientes para avatares de membros)
2. **`AdminDashboardClient.tsx`** — Gradientes para cards de produtos no dashboard
3. **`ProductCard.tsx`** (member) — 6 gradientes dark para fallback quando produto não tem banner

### Decisões Técnicas

- Arquivo de constantes puro (sem lógica, apenas definições)
- Manter funções de seleção (hash-based, index-based) nos componentes que as usam
- Usar `as const` para type safety (readonly tuples)
- Absolute import: `@/lib/constants/gradients`
- Se a Story 11.5 já tiver criado tokens para as cores dos gradientes, usar os nomes Tailwind dos tokens em vez de hex hardcoded

### Estrutura Proposta

```typescript
// src/lib/constants/gradients.ts

export const AVATAR_GRADIENTS = [
  'from-[#2563EB] to-[#7C3AED]',
  'from-[#059669] to-[#0EA5E9]',
  // ... extrair exatamente o que está nos componentes atuais
] as const;

export const PRODUCT_GRADIENTS = [
  // ... extrair do AdminDashboardClient
] as const;

export const BANNER_FALLBACK_GRADIENTS = [
  // ... extrair do ProductCard
] as const;
```

### Testing

- **Framework:** vitest
- **Abordagem:** Não há testes unitários para essa story (extração de constantes, sem lógica). Validação via:
  - `npm run typecheck` — garante que imports e tipos estão corretos
  - `npm run build` — garante compilação sem erros
  - Revisão visual — verificar que gradientes renderizam identicamente no browser

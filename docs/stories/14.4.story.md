## Story

**Como** administrador da plataforma,
**Quero** habilitar ou desabilitar a emissão de certificados por produto,
**Para que** eu possa controlar quais cursos oferecem certificado de conclusão aos membros.

## Acceptance Criteria

1. **AC1** — Coluna `certificate_enabled` (boolean, default `false`) adicionada à tabela `products` via migration.
2. **AC2** — Type definitions em `src/types/database.ts` atualizadas com o campo `certificate_enabled` no tipo `products`.
3. **AC3** — Na página de edição do produto no admin (`/admin/products/[id]`), exibe um toggle/switch "Emitir certificado de conclusão" na seção de configurações.
4. **AC4** — O toggle reflete o estado atual de `certificate_enabled` do produto e permite alternar.
5. **AC5** — Ao salvar o produto (PUT/PATCH existente), o campo `certificate_enabled` é persistido no banco.
6. **AC6** — O endpoint `PUT /api/products/[id]` (ou PATCH) aceita e persiste o campo `certificate_enabled`.
7. **AC7** — Produtos novos são criados com `certificate_enabled = false` por default.
8. **AC8** — A API `POST /api/certificates/generate` (Story 14.2) respeita o campo: se `certificate_enabled = false`, retorna erro `CERTIFICATES_DISABLED` (403).

### Migration DDL

```sql
-- Add certificate_enabled column to products
ALTER TABLE products
ADD COLUMN certificate_enabled BOOLEAN NOT NULL DEFAULT FALSE;

-- Comment for documentation
COMMENT ON COLUMN products.certificate_enabled IS 'Whether completion certificates are enabled for this product';
```

### Toggle Component Pattern

```tsx
'use client';

interface CertificateToggleProps {
  enabled: boolean;
  onChange: (enabled: boolean) => void;
}

export function CertificateToggle({ enabled, onChange }: CertificateToggleProps) {
  return (
    <div className="flex items-center justify-between p-4 bg-dark-surface rounded-lg border border-white/10">
      <div>
        <h3 className="text-sm font-medium text-white">Certificado de Conclusão</h3>
        <p className="text-xs text-gray-400 mt-1">
          Membros poderão baixar um certificado ao completar 100% do curso
        </p>
      </div>
      <button
        type="button"
        role="switch"
        aria-checked={enabled}
        onClick={() => onChange(!enabled)}
        className={`relative inline-flex h-6 w-11 items-center rounded-full transition-colors ${
          enabled ? 'bg-primary' : 'bg-gray-600'
        }`}
      >
        <span className={`inline-block h-4 w-4 transform rounded-full bg-white transition-transform ${
          enabled ? 'translate-x-6' : 'translate-x-1'
        }`} />
      </button>
    </div>
  );
}
```

### Integration with Certificate Generation API (Story 14.2)

No `POST /api/certificates/generate`, adicionar check antes de gerar:

```typescript
const { data: product } = await supabase
  .from('products')
  .select('id, title, certificate_enabled')
  .eq('id', productId)
  .single();

if (!product?.certificate_enabled) {
  return apiError('CERTIFICATES_DISABLED', 'Certificados não estão habilitados para este produto', 403);
}
```

### Edge Cases

- **Produto existente sem campo:** A migration com `DEFAULT FALSE` garante que todos os produtos existentes têm certificado desabilitado.
- **Admin desabilita depois de gerar:** Certificados já gerados continuam acessíveis. Apenas novos certificados são bloqueados.
- **Product POST (criação):** Já funciona com default `false` — não precisa enviar o campo.

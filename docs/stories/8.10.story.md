## Story

**As a** desenvolvedor,
**I want** cachear o role do usuário no JWT custom claims do Supabase e adicionar rate limiting nos endpoints de webhook,
**so that** o middleware não faça query no banco em toda request e webhooks fiquem protegidos contra abuso.

---

## Context

### Estado Atual

O middleware de autenticação (provavelmente em `middleware.ts` ou equivalente) verifica o role do usuário fazendo query na tabela `profiles` em toda request a rotas `/admin`. Isso gera uma query extra por request.

O endpoint de webhook (`POST /api/webhooks/payt/`) não tem rate limiting — qualquer IP pode enviar requests ilimitadamente.

### Mudança Necessária

1. **JWT Claims**: Criar trigger no Supabase que sincroniza `profiles.role` com `auth.users.raw_app_meta_data.role`. Middleware lê do JWT em vez de fazer query.
2. **Rate Limiting**: Implementar rate limiter in-memory (Map com TTL) no webhook endpoint — 30 req/min por IP, retorna 429 quando excedido.

### Decisões Técnicas

- Trigger SQL com `SECURITY DEFINER` para acessar `auth.users` (tabela do schema `auth`)
- Backfill necessário para profiles existentes
- Middleware mantém fallback para query em `profiles` se `app_metadata.role` não existir (backward compat durante rollout)
- Rate limiter usa `Map<string, {count, resetAt}>` — sem Redis ou dependência externa. Cleanup periódico de entries expiradas
- Rate limiter é singleton (module-level variable) — persiste entre requests no mesmo worker

---

## Acceptance Criteria

1. Function SQL `set_user_role(user_id uuid, new_role text)` que atualiza `raw_app_meta_data.role` em `auth.users`
2. Trigger `on_profile_role_change` em `profiles`: AFTER UPDATE OF role → chama `set_user_role`
3. Backfill: todos os profiles existentes têm `app_metadata.role` sincronizado
4. Middleware de admin lê `user.app_metadata?.role` primeiro; se indisponível, faz fallback para query em `profiles`
5. Rate limiting no `POST /api/webhooks/payt`: máximo **30 requests por minuto por IP**
6. Rate limiter implementado via **in-memory Map** com TTL — sem dependência externa
7. Quando excedido, retorna **429 Too Many Requests** com header `Retry-After` (segundos até reset)
8. Rate limiter faz cleanup de entries expiradas periodicamente (a cada 100 checks ou similar)
9. Build, typecheck e lint passam

## Dev Notes

### SQL para custom claims

```sql
-- Function para atualizar role no JWT
CREATE OR REPLACE FUNCTION public.set_user_role(user_id uuid, new_role text)
RETURNS void AS $$
BEGIN
  UPDATE auth.users
  SET raw_app_meta_data = COALESCE(raw_app_meta_data, '{}'::jsonb) || jsonb_build_object('role', new_role)
  WHERE id = user_id;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

-- Trigger function
CREATE OR REPLACE FUNCTION public.handle_profile_role_change()
RETURNS trigger AS $$
BEGIN
  IF NEW.role IS DISTINCT FROM OLD.role THEN
    PERFORM public.set_user_role(NEW.id, NEW.role);
  END IF;
  RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

-- Trigger
CREATE TRIGGER on_profile_role_change
  AFTER UPDATE OF role ON public.profiles
  FOR EACH ROW
  EXECUTE FUNCTION public.handle_profile_role_change();

-- Backfill
DO $$
DECLARE r RECORD;
BEGIN
  FOR r IN SELECT id, role FROM public.profiles LOOP
    PERFORM public.set_user_role(r.id, r.role);
  END LOOP;
END;
$$;
```

### Rate Limiter

```typescript
// src/lib/utils/rate-limiter.ts
export class RateLimiter {
  private store = new Map<string, { count: number; resetAt: number }>();
  private checkCount = 0;

  constructor(
    private maxRequests = 30,
    private windowMs = 60_000
  ) {}

  check(key: string): { allowed: boolean; retryAfter?: number } {
    this.checkCount++;
    if (this.checkCount % 100 === 0) this.cleanup();

    const now = Date.now();
    const entry = this.store.get(key);

    if (!entry || now > entry.resetAt) {
      this.store.set(key, { count: 1, resetAt: now + this.windowMs });
      return { allowed: true };
    }

    if (entry.count < this.maxRequests) {
      entry.count++;
      return { allowed: true };
    }

    return { allowed: false, retryAfter: Math.ceil((entry.resetAt - now) / 1000) };
  }

  private cleanup() {
    const now = Date.now();
    for (const [key, entry] of this.store) {
      if (now > entry.resetAt) this.store.delete(key);
    }
  }
}
```

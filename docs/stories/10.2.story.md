## Story

**As a** member/admin,
**I want** ver uma página de erro amigável quando algo falha em vez de uma tela branca,
**so that** eu saiba que algo deu errado e possa tentar novamente sem perder o contexto.

---

## Context

### Estado Atual

O Brownfield Assessment identificou que nenhuma rota possui `error.tsx` (Next.js error boundary). Consequências:

1. **Member dashboard** (`(member)/page.tsx`) — `Promise.all` sem try/catch. Se qualquer query falha, a página inteira crasheia com tela branca.
2. **Admin dashboard** (`admin/page.tsx`) — Mesmo problema: `Promise.all()` sem error handling.
3. **Lesson player** — Se fetch da lição falha, renderiza layout vazio sem feedback.

### Mudança Necessária

1. Criar `error.tsx` no route group `(member)/` com UI de fallback
2. Criar `error.tsx` no route group `admin/` com UI de fallback
3. Criar `not-found.tsx` global (app root) para 404 customizado
4. Adicionar try/catch em `(member)/page.tsx` e `admin/page.tsx` com fallback graceful
5. Garantir que falha em uma query não derruba todo o dashboard (queries independentes com `Promise.allSettled`)

### Decisões Técnicas

- Error pages usam design system existente (dark theme, Netflix-style)
- Botão "Tentar novamente" chama `reset()` do Next.js error boundary
- `Promise.allSettled()` em dashboards: métricas que falham mostram "—" em vez de crashar tudo
- Error pages são client components (`'use client'`) conforme exigido pelo Next.js

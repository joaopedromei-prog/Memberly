## Story

**As a** membro,
**I want** visualizar os módulos de um produto em um grid de cards retrato (600×840px) com banners visuais,
**so that** eu tenha uma experiência visual imersiva e estilo Netflix ao navegar pelos módulos do curso.

---

## Context

### Referência Visual

A imagem de referência mostra:
- Fundo escuro (dark theme)
- Título "Módulos" no topo
- Grid de 4 colunas em desktop
- Cards em formato **retrato** (proporção ~5:7 / 600×840px)
- Banners ocupam o card inteiro como background
- Sem barra de progresso visível nos cards
- Hover effect sutil nos cards
- Responsivo: 4 colunas → 3 → 2 → 1 conforme breakpoint

### Estado Atual

Atualmente os módulos são exibidos na página `products/[slug]` usando `ModuleList` + `ModuleCard` com layout **horizontal** (thumbnail + texto ao lado). O design é funcional mas não visual/imersivo como a referência.

### Mudança Necessária

Refatorar `ModuleCard` e `ModuleList` na página de detalhes do produto para usar o layout em grid com cards retrato, mantendo a funcionalidade existente (progresso, navegação).

---

## Acceptance Criteria

1. Na página de detalhes do produto (`/products/[slug]`), os módulos são exibidos em um **grid de cards retrato** com proporção aspect-ratio 5:7 (equivalente a 600×840px)
2. Cada card exibe o **banner do módulo como imagem de fundo** ocupando todo o card
3. O grid é responsivo: **4 colunas** em desktop (≥1280px), **3 colunas** em tablet (≥768px), **2 colunas** em mobile (≥640px), **1 coluna** abaixo
4. Módulos sem banner exibem um **placeholder visual** (gradiente escuro com título centralizado)
5. Ao clicar em um card, o membro navega para a **próxima aula não concluída** do módulo (comportamento existente mantido)
6. O título do módulo aparece na **parte inferior do card** com overlay gradiente escuro para legibilidade
7. Um **indicador discreto de progresso** aparece no card (ex: barra fina no bottom ou badge de "X/Y aulas")
8. O card tem **efeito hover** sutil (scale + brilho/border)
9. A seção mantém o título "Módulos" acima do grid
10. A funcionalidade de admin preview mode continua operacional

## Dev Notes

### Proporção do Card

```
Aspect ratio: 5/7 (600px × 840px)
CSS: aspect-[5/7]
```

### Estrutura do Card (referência)

```tsx
<Link href={href}>
  <article className="group relative aspect-[5/7] overflow-hidden rounded-lg">
    {/* Banner Image (fill) */}
    <Image src={bannerUrl} alt={...} fill className="object-cover" />

    {/* Gradient overlay bottom */}
    <div className="absolute inset-0 bg-gradient-to-t from-black/80 via-black/20 to-transparent" />

    {/* Title + progress at bottom */}
    <div className="absolute bottom-0 left-0 right-0 p-4">
      <h3 className="text-lg font-bold text-white">{title}</h3>
      <p className="text-xs text-neutral-300">{completedLessons}/{totalLessons} aulas</p>
    </div>
  </article>
</Link>
```

### Grid Layout

```tsx
<div className="grid grid-cols-1 gap-4 sm:grid-cols-2 md:grid-cols-3 xl:grid-cols-4">
  {modules.map(mod => <ModuleCard key={mod.id} ... />)}
</div>
```

### Efeito Hover

```css
group hover:scale-105 transition-transform duration-300
hover:ring-1 hover:ring-white/20
```

### Dimensões de Banner Recomendadas

- Upload/geração de banners de módulo: **600×840px** (proporção 5:7)
- O campo `banner_url` já existe na tabela `modules` e no componente

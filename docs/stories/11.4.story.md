## Dev Notes

### Context

Existem 6 modals/dialogs no admin (+ `ConfirmDialog` em ui/), cada um reimplementando a mesma estrutura:

```tsx
// Padrão repetido em TODOS os dialogs:
<div className="fixed inset-0 z-50 flex items-center justify-center">
  <div className="fixed inset-0 bg-black/50" onClick={onClose} />
  <div className="relative z-10 w-full max-w-md rounded-lg bg-white p-6 shadow-xl">
    {/* header, content, actions */}
  </div>
</div>
```

### API Proposta

```typescript
interface DialogProps {
  open: boolean;
  onClose: () => void;
  size?: 'sm' | 'md' | 'lg' | 'xl';
  children: React.ReactNode;
}

// Compound components:
Dialog.Header: React.FC<{ children: React.ReactNode }>
Dialog.Body: React.FC<{ children: React.ReactNode }>
Dialog.Footer: React.FC<{ children: React.ReactNode }>

// Uso:
<Dialog open={isOpen} onClose={handleClose} size="md">
  <Dialog.Header>Título</Dialog.Header>
  <Dialog.Body>Conteúdo</Dialog.Body>
  <Dialog.Footer>
    <Button variant="outline" onClick={handleClose}>Cancelar</Button>
    <Button onClick={handleConfirm}>Confirmar</Button>
  </Dialog.Footer>
</Dialog>
```

### Decisões Técnicas

- Compound component pattern (mesmo usado em `SlideOver.tsx`)
- NÃO usar React Portal nesta versão — manter simples com fixed positioning
- `SlideOver.tsx` é padrão diferente (lateral) — NÃO deve ser afetado
- Cada dialog refatorado deve manter sua API pública inalterada (props)
- Focus trap seria ideal mas é escopo extra — implementar apenas Escape key e click-outside

### Testing

- **Framework:** vitest + @testing-library/react + jsdom
- **Localização:** `packages/memberly-app/tests/unit/components/ui/Dialog.test.tsx` (novo)
- **Setup:** `tests/unit/setup.ts` (já existente)
- **Cobertura necessária:**
  - Renderização condicional (open prop)
  - Keyboard events (Escape)
  - Mouse events (click overlay vs click content)
  - Sub-componentes renderizam corretamente
  - Atributos ARIA presentes

### Referências

- `ConfirmDialog.tsx` como referência de padrão atual
- `SlideOver.tsx` como exemplo de compound component existente no projeto

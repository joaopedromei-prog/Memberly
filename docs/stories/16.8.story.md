## Story

**Como** membro da plataforma,
**Quero** receber uma notificacao in-app quando desbloqueio um novo badge,
**Para que** eu saiba imediatamente sobre minha conquista e me sinta recompensado.

## Acceptance Criteria

1. **AC1** — Novo tipo de notificacao `BADGE_UNLOCKED` adicionado a `NOTIFICATION_TYPES` em `src/lib/notifications/types.ts`.
2. **AC2** — Tipo `BADGE_UNLOCKED` adicionado a `NotificationType` union type.
3. **AC3** — Trigger `src/lib/notifications/triggers/badge-unlocked.ts` exporta funcao `notifyBadgeUnlocked(profileId: string, badgeId: string): Promise<void>`.
4. **AC4** — O trigger busca o badge pelo `badgeId` para obter nome e descricao. Titulo: "Nova conquista desbloqueada!". Body: "Voce conquistou: {badgeName}". Data: `{ badgeId, badgeName }`.
5. **AC5** — O trigger e chamado dentro de `evaluateBadges()` (badge-engine.ts) para cada badge recem-desbloqueado, usando fire-and-forget pattern.
6. **AC6** — Falha na notificacao NAO impede o desbloqueio do badge.
7. **AC7** — Testes unitarios para o trigger `notifyBadgeUnlocked`.

### Badge Unlocked Trigger (seguir pattern de triggers existentes)

```typescript
// src/lib/notifications/triggers/badge-unlocked.ts
import { createAdminClient } from '@/lib/supabase/admin';
import { createNotification } from '../create-notification';

export async function notifyBadgeUnlocked(
  profileId: string,
  badgeId: string
): Promise<void> {
  const adminClient = createAdminClient();

  // Get badge info
  const { data: badge } = await adminClient
    .from('badges')
    .select('name, description')
    .eq('id', badgeId)
    .single();

  if (!badge) return;

  await createNotification({
    profileId,
    type: 'BADGE_UNLOCKED',
    title: 'Nova conquista desbloqueada!',
    body: `Voce conquistou: ${badge.name}`,
    data: { badgeId, badgeName: badge.name },
  });
}
```

### Integration in Badge Engine (fire-and-forget)

```typescript
// In src/lib/gamification/badge-engine.ts — inside evaluateBadges()

```

### Fire-and-Forget Pattern (identico a 15.3)

```typescript
notifyBadgeUnlocked(profileId, badge.id).catch(() => {/* silent */});
```

- Falha na notificacao e silenciosa
- Nao bloqueia o desbloqueio do badge
- Nao afeta o retorno de `evaluateBadges()`

### Edge Cases

- **Badge deletado entre unlock e notify:** `notifyBadgeUnlocked` retorna silenciosamente se badge nao encontrado.
- **Notificacao desabilitada:** `createNotification` ja verifica `notification_preferences` — se membro desabilitou `BADGE_UNLOCKED`, nao cria.
- **Multiplos badges desbloqueados:** Cada badge gera sua propria notificacao independente.

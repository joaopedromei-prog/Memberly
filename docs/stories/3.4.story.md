## Story

**As a** membro,
**I want** comentar nas aulas para tirar dúvidas,
**so that** eu possa interagir e esclarecer pontos do conteúdo.

---

## Acceptance Criteria

1. Seção de comentários abaixo do player na página de aula
2. Textarea para novo comentário com botão "Enviar"
3. Lista de comentários em ordem cronológica (mais recentes primeiro) com: nome do autor, avatar placeholder, data relativa ("há 2 horas"), conteúdo
4. Suporte a replies (1 nível): botão "Responder" em cada comentário, replies exibidos indentados abaixo do pai
5. Admin pode responder comentários (identificado com badge "Admin")
6. Paginação ou load-more para comentários (preparado para muitas interações)
7. Validação: comentário não pode ser vazio, limite de 2000 caracteres
8. Membro só pode comentar em aulas de produtos que tem acesso

### Route Handler Implementation Pattern

**GET `/api/lessons/[id]/comments/route.ts`:**
```typescript
import { createServerSupabaseClient } from '@/lib/supabase/server';
import { NextRequest, NextResponse } from 'next/server';

export async function GET(
  request: NextRequest,
  { params }: { params: { id: string } }
) {
  const supabase = await createServerSupabaseClient();
  const { searchParams } = new URL(request.url);
  const cursor = searchParams.get('cursor'); // ISO date string
  const limit = 10;

  let query = supabase
    .from('comments')
    .select(`
      id, content, parent_id, created_at,
      profile:profiles ( id, full_name, avatar_url, role ),
      replies:comments (
        id, content, created_at,
        profile:profiles ( id, full_name, avatar_url, role )
      )
    `)
    .eq('lesson_id', params.id)
    .is('parent_id', null)  // Only top-level comments
    .order('created_at', { ascending: false })
    .limit(limit);

  if (cursor) {
    query = query.lt('created_at', cursor);
  }

  const { data, error } = await query;
  // ...return response
}
```

**POST `/api/lessons/[id]/comments/route.ts`:**
```typescript
export async function POST(
  request: NextRequest,
  { params }: { params: { id: string } }
) {
  const supabase = await createServerSupabaseClient();
  const { data: { user } } = await supabase.auth.getUser();
  const body = await request.json();

  // Validate
  if (!body.content?.trim()) {
    return NextResponse.json(
      { error: { code: 'VALIDATION_ERROR', message: 'Comentário não pode ser vazio' } },
      { status: 400 }
    );
  }
  if (body.content.length > 2000) {
    return NextResponse.json(
      { error: { code: 'VALIDATION_ERROR', message: 'Limite de 2000 caracteres' } },
      { status: 400 }
    );
  }

  const { data, error } = await supabase
    .from('comments')
    .insert({
      lesson_id: params.id,
      profile_id: user.id,
      content: body.content.trim(),
      parent_id: body.parent_id || null
    })
    .select('id, content, created_at, profile:profiles ( id, full_name, avatar_url, role )')
    .single();
  // ...return response
}
```

### Database Schema [Source: architecture.md#8-database-schema]

```sql
-- Comments table
CREATE TABLE comments (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    lesson_id UUID NOT NULL REFERENCES lessons(id) ON DELETE CASCADE,
    profile_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
    content TEXT NOT NULL CHECK (char_length(content) <= 2000),
    parent_id UUID REFERENCES comments(id) ON DELETE CASCADE,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- RLS: Members can comment on lessons of products they have access to
-- RLS: Members can view comments on lessons they can access
```

### Accessibility [Source: front-end-spec.md#7-accessibility-requirements]

- Textarea: `<label>` visível "Escreva seu comentário" (ou sr-only)
- Botão "Enviar": `aria-label="Enviar comentário"`, disabled state communicated
- Comments list: `role="list"`, cada comment `role="listitem"`
- Replies: nested `role="list"` dentro do parent
- "Responder": `aria-expanded` toggle quando form aberto
- Badge Admin: `aria-label="Administrador"` no badge
- Contador de caracteres: `aria-live="polite"` para anunciar mudanças
- Focus management: após submit, foco volta ao textarea

## Story

**Como** membro da plataforma,
**Quero** que meu streak de dias consecutivos seja atualizado automaticamente quando eu completo uma aula,
**Para que** eu possa acompanhar minha consistencia de estudo e desbloquear badges de streak.

## Acceptance Criteria

1. **AC1** — Servico `src/lib/gamification/streak-tracker.ts` exporta funcao `updateStreak(profileId: string): Promise<void>`.
2. **AC2** — Se `last_activity_date` = hoje (UTC): no-op (ja registrou atividade hoje).
3. **AC3** — Se `last_activity_date` = ontem (UTC): incrementa `current_streak` em 1.
4. **AC4** — Se `last_activity_date` e anterior a ontem ou null: reseta `current_streak` para 1 (novo streak).
5. **AC5** — Sempre atualiza `longest_streak = MAX(current_streak, longest_streak)`.
6. **AC6** — Sempre atualiza `last_activity_date` para hoje (UTC) e `updated_at` para NOW().
7. **AC7** — Se nao existe registro em `streaks` para o membro, cria um novo com `current_streak = 1`.
8. **AC8** — Usa admin client (service role) para bypass de RLS.
9. **AC9** — Todas as comparacoes de data usam UTC (date part only, sem timezone issues).
10. **AC10** — Integrado no `POST /api/progress/[lessonId]` — disparar quando membro marca aula como completed. Usar fire-and-forget pattern.
11. **AC11** — Testes unitarios para cada cenario de streak (no-op, increment, reset, novo registro).

### Streak Tracker Implementation

```typescript
// src/lib/gamification/streak-tracker.ts
import { createAdminClient } from '@/lib/supabase/admin';

export async function updateStreak(profileId: string): Promise<void> {
  const adminClient = createAdminClient();

  // Get current date in UTC (date only, no time)
  const today = new Date().toISOString().split('T')[0]; // 'YYYY-MM-DD'
  const yesterday = new Date(Date.now() - 86400000).toISOString().split('T')[0];

  // Get existing streak
  const { data: streak } = await adminClient
    .from('streaks')
    .select('*')
    .eq('profile_id', profileId)
    .maybeSingle();

  if (!streak) {
    // No streak record — create new
    await adminClient.from('streaks').insert({
      profile_id: profileId,
      current_streak: 1,
      longest_streak: 1,
      last_activity_date: today,
    });
    return;
  }

  // Already registered today — no-op
  if (streak.last_activity_date === today) return;

  let newStreak: number;

  if (streak.last_activity_date === yesterday) {
    // Consecutive day — increment
    newStreak = streak.current_streak + 1;
  } else {
    // Gap — reset
    newStreak = 1;
  }

  const newLongest = Math.max(newStreak, streak.longest_streak);

  await adminClient
    .from('streaks')
    .update({
      current_streak: newStreak,
      longest_streak: newLongest,
      last_activity_date: today,
      updated_at: new Date().toISOString(),
    })
    .eq('profile_id', profileId);
}
```

### Edge Cases

- **Timezone:** Usar UTC para evitar problemas de timezone. O client pode exibir "streak atualizado" com base na data local, mas o server sempre opera em UTC.
- **Multiplas aulas no mesmo dia:** Apenas o primeiro `updateStreak` do dia tem efeito; chamadas subsequentes sao no-op (AC2).
- **Membro sem streak record:** Criar automaticamente na primeira atividade (AC7).
- **Gap de 1 dia:** Se membro pula 1 dia, streak reseta para 1 no proximo dia de atividade.
- **Concorrencia:** Se duas chamadas concorrentes acontecem, a segunda sera no-op (last_activity_date ja sera hoje apos a primeira).

### Testing

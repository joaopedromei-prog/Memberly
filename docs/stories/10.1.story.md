## Story

**As an** admin/member,
**I want** que conteúdo HTML seja sanitizado antes de renderização e que todas as rotas admin exijam autorização,
**so that** atacantes não consigam injetar scripts maliciosos via descrições de lições ou templates de email, e usuários comuns não consigam executar operações admin.

---

## Context

### Estado Atual

O Brownfield Assessment (2026-03-13) identificou 4 vulnerabilidades de segurança:

1. **XSS em `EmailSettings.tsx`** — `dangerouslySetInnerHTML` renderiza template de email sem sanitização. Um admin que insere `<script>` no template compromete qualquer usuário que visualize o preview.

2. **XSS em `LessonInfo.tsx`** — `dangerouslySetInnerHTML` renderiza descrição de lição diretamente do banco. Se conteúdo malicioso for salvo via API (bypass da UI), executa no browser de todos os membros.

3. **Auth bypass em `PATCH /api/products/[id]/modules/reorder`** — Falta `requireAdmin()`. Qualquer usuário autenticado pode reordenar módulos de qualquer produto.

4. **Auth bypass em rotas de member access** — Falta revalidação de admin em certos métodos HTTP.

### Mudança Necessária

1. Instalar `dompurify` (+ `@types/dompurify`) como dependência
2. Criar utility `src/lib/utils/sanitize.ts` que exporta `sanitizeHtml(html: string): string`
3. Sanitizar HTML em `EmailSettings.tsx` antes de `dangerouslySetInnerHTML`
4. Sanitizar HTML em `LessonInfo.tsx` antes de `dangerouslySetInnerHTML`
5. Adicionar `requireAdmin()` no handler PATCH de `src/app/api/products/[id]/modules/reorder/route.ts`
6. Revisar e adicionar `requireAdmin()` em todas as rotas de mutação dentro de `src/app/api/` que ainda não têm

### Decisões Técnicas

- Usar `dompurify` (biblioteca mais madura e auditada para sanitização XSS)
- Sanitizar no momento da renderização (não no save) para proteção retroativa
- `sanitizeHtml()` permite apenas tags seguras: `<p>`, `<br>`, `<strong>`, `<em>`, `<ul>`, `<ol>`, `<li>`, `<a>`, `<h1>`-`<h6>`, `<blockquote>`, `<code>`
- Links `<a>` devem ter `target="_blank"` e `rel="noopener noreferrer"` forçados

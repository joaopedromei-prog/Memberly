## Story

**As an** admin,
**I want** configurar módulos e aulas para ficarem disponíveis somente após X dias da data de compra do membro,
**so that** eu possa controlar a liberação progressiva do conteúdo (drip content) e manter o engajamento ao longo do tempo.

**As a** membro,
**I want** ver um popup elegante quando tento acessar conteúdo ainda não liberado, informando exatamente quando será disponibilizado,
**so that** eu saiba que o conteúdo existe e quando poderei acessá-lo.

---

## Context

### Estado Atual

Todos os módulos e aulas ficam disponíveis imediatamente após o membro receber acesso ao produto. Não existe conceito de liberação progressiva. O campo `granted_at` em `member_access` já registra quando o acesso foi concedido.

### Mudança Necessária

Adicionar campo `drip_days` em `modules` e `lessons`. Na área do membro, conteúdo com drip ativo aparece bloqueado (cadeado + opacidade) com popup de countdown ao clicar. Admin preview ignora drip.

### Decisões Técnicas

- `drip_days` é nullable integer: `null` = disponível imediatamente
- Herança: aula herda o drip do módulo pai, mas pode ter override. Regra: `effectiveDripDays = Math.max(module.drip_days ?? 0, lesson.drip_days ?? 0)` — aula nunca pode ser mais permissiva que o módulo
- Cálculo baseado em `member_access.granted_at`: `unlockDate = granted_at + drip_days * 86400000ms`
- Countdown atualiza a cada 60 segundos via `setInterval`
- Popup é um toast fixo no canto inferior direito (não um modal — menos invasivo)

## Acceptance Criteria

1. Campo `drip_days` (integer, nullable) adicionado nas tabelas `modules` e `lessons`. Quando `null` = disponível imediatamente
2. No form de módulo (admin), campo "Disponível após X dias da compra" — input numérico, opcional, com helper text
3. No form de aula (admin), campo "Disponível após X dias da compra" — input numérico, opcional. Se o módulo pai tem drip_days, exibe nota: "Mínimo herdado do módulo: X dias"
4. Na área do membro, módulos/aulas com drip ativo exibem ícone de **cadeado** e **opacidade reduzida** (não clicáveis para navegação normal)
5. Ao clicar em conteúdo bloqueado, exibe toast fixo no canto inferior direito: "Este conteúdo estará disponível em X dias, Y horas e Z minutos"
6. O cálculo usa `member_access.granted_at` como base: conteúdo liberado quando `now() >= granted_at + drip_days * 24h`
7. Countdown no toast atualiza em **tempo real** (a cada 60 segundos)
8. Toast tem botão de fechar (X)
9. Admin preview mode **ignora drip** — mostra tudo desbloqueado
10. Build, typecheck e lint passam

## Dev Notes

### Cálculo de Drip

```typescript
// src/lib/utils/drip.ts

export function isDripUnlocked(grantedAt: string, dripDays: number | null): boolean {
  if (dripDays === null || dripDays === 0) return true;
  const unlockDate = new Date(grantedAt).getTime() + dripDays * 86_400_000;
  return Date.now() >= unlockDate;
}

export function getEffectiveDripDays(
  moduleDripDays: number | null,
  lessonDripDays: number | null
): number {
  return Math.max(moduleDripDays ?? 0, lessonDripDays ?? 0);
}

export function getDripCountdown(
  grantedAt: string,
  dripDays: number
): { days: number; hours: number; minutes: number } {
  const unlockDate = new Date(grantedAt).getTime() + dripDays * 86_400_000;
  const diff = Math.max(0, unlockDate - Date.now());
  return {
    days: Math.floor(diff / 86_400_000),
    hours: Math.floor((diff % 86_400_000) / 3_600_000),
    minutes: Math.floor((diff % 3_600_000) / 60_000),
  };
}
```

### Herança módulo → aula

| Módulo drip_days | Aula drip_days | Effective | Razão |
|------------------|----------------|-----------|-------|
| null | null | 0 (disponível) | Sem drip |
| 7 | null | 7 | Herda do módulo |
| 7 | 14 | 14 | Override da aula (maior) |
| 7 | 3 | 7 | Módulo é o piso — aula não pode ser menor |
| null | 5 | 5 | Aula define sozinha |

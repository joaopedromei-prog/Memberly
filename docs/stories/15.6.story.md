## Story

**Como** administrador da plataforma,
**Quero** poder habilitar ou desabilitar tipos de notificação por produto na área de configurações do produto,
**Para que** eu possa controlar quais notificações os membros recebem para cada curso, evitando notificação desnecessária.

## Acceptance Criteria

1. **AC1** — Migration adiciona coluna `notifications_config` (JSONB, default `'{"NEW_LESSON": true, "COMMENT_REPLY": true, "COURSE_COMPLETED": true}'::jsonb`) na tabela `products`. Migration: `20260320000003_add_notifications_config_to_products.sql`.
2. **AC2** — Type `products` em `database.ts` atualizado com campo `notifications_config` do tipo `{ NEW_LESSON: boolean; COMMENT_REPLY: boolean; COURSE_COMPLETED: boolean }`.
3. **AC3** — Componente `src/components/admin/NotificationSettings.tsx` exibe toggles para cada tipo de notificação (Nova aula, Resposta a comentário, Curso concluído) dentro da página de edição do produto admin.
4. **AC4** — `PATCH /api/products/[id]` aceita `notifications_config` no body e atualiza o campo no banco.
5. **AC5** — O serviço de criação de notificações (15.3) verifica `products.notifications_config` antes de criar notificações: se o tipo está `false` para o produto, não cria a notificação.
6. **AC6** — Quando `notifications_config` não existe no produto (produtos criados antes da migration), assume todos como `true` (comportamento default).
7. **AC7** — Testes unitários para o componente NotificationSettings e integração com a API.

### NotificationSettings Component Pattern (seguir CertificateToggle)

```tsx
'use client';

import { useState } from 'react';
import { Bell, BookOpen, MessageCircle, Trophy } from 'lucide-react';

interface NotificationsConfig {
  NEW_LESSON: boolean;
  COMMENT_REPLY: boolean;
  COURSE_COMPLETED: boolean;
}

interface Props {
  productId: string;
  initialConfig: NotificationsConfig;
}

const NOTIFICATION_LABELS = [
  { key: 'NEW_LESSON' as const, label: 'Nova aula publicada', description: 'Notificar membros quando uma nova aula é publicada', icon: BookOpen },
  { key: 'COMMENT_REPLY' as const, label: 'Resposta a comentário', description: 'Notificar quando alguém responde a um comentário', icon: MessageCircle },
  { key: 'COURSE_COMPLETED' as const, label: 'Curso concluído', description: 'Notificar quando o membro completa 100% do curso', icon: Trophy },
];

export function NotificationSettings({ productId, initialConfig }: Props) {
  const [config, setConfig] = useState<NotificationsConfig>(initialConfig);
  const [saving, setSaving] = useState(false);

  async function handleToggle(key: keyof NotificationsConfig) {
    const newConfig = { ...config, [key]: !config[key] };
    setConfig(newConfig);
    setSaving(true);

    await fetch(`/api/products/${productId}`, {
      method: 'PATCH',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ notifications_config: newConfig }),
    });

    setSaving(false);
  }

  return (
    <div className="bg-dark-surface rounded-lg p-6">
      <div className="flex items-center gap-2 mb-4">
        <Bell size={20} className="text-primary" />
        <h3 className="text-lg font-semibold text-white">Notificações</h3>
      </div>
      <div className="space-y-4">
        {NOTIFICATION_LABELS.map(({ key, label, description, icon: Icon }) => (
          <div key={key} className="flex items-center justify-between">
            <div className="flex items-center gap-3">
              <Icon size={16} className="text-neutral-400" />
              <div>
                <p className="text-sm text-white">{label}</p>
                <p className="text-xs text-neutral-500">{description}</p>
              </div>
            </div>
            <button
              onClick={() => handleToggle(key)}
              className={`relative w-11 h-6 rounded-full transition-colors ${
                config[key] ? 'bg-primary' : 'bg-dark-card'
              }`}
              disabled={saving}
            >
              <span className={`absolute top-1 left-1 w-4 h-4 rounded-full bg-white transition-transform ${
                config[key] ? 'translate-x-5' : 'translate-x-0'
              }`} />
            </button>
          </div>
        ))}
      </div>
    </div>
  );
}
```

### Trigger Integration Pattern

Modificar cada trigger para verificar o config do produto:

```typescript
// In new-lesson.ts, comment-reply.ts, course-completed.ts
const { data: product } = await adminClient
  .from('products')
  .select('notifications_config')
  .eq('id', productId)
  .single();

const config = product?.notifications_config as NotificationsConfig | null;

// Default to true if config is null (pre-migration products) or key doesn't exist
const isEnabled = config?.[notificationType] ?? true;
if (!isEnabled) return;
```

### PATCH /api/products/[id] — Accept notifications_config

O endpoint existente já aceita campos dinâmicos. Verificar se `notifications_config` é incluído no objeto de update. Se o endpoint usa allow-list de campos, adicionar `notifications_config`:

```typescript
if (body.notifications_config !== undefined) {
  updates.notifications_config = body.notifications_config;
}
```

### Edge Cases

- **Produtos existentes sem coluna:** A migration com DEFAULT garante que todos os produtos existentes recebem o valor padrão `{ NEW_LESSON: true, COMMENT_REPLY: true, COURSE_COMPLETED: true }`.
- **JSONB parcial:** Se admin salvar apenas `{ NEW_LESSON: false }` sem os outros campos, os triggers devem usar `?? true` para campos ausentes.
- **Validação do body:** Validar que `notifications_config` contém apenas chaves válidas e valores boolean antes de salvar.
- **Otimistic update:** O toggle muda estado local imediatamente, com save em background. Se falhar, reverter.
- **Concorrência:** Dois admins editando ao mesmo tempo — last-write-wins, aceitável para v1.

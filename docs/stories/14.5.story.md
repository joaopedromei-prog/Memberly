## Story

**Como** qualquer pessoa com o link de validação de um certificado,
**Quero** acessar uma página pública que exibe os dados do certificado,
**Para que** eu possa verificar a autenticidade do certificado de conclusão de um membro.

## Acceptance Criteria

1. **AC1** — Página pública acessível em `/certificates/[hash]` sem autenticação (rota pública).
2. **AC2** — A página consulta a tabela `certificates` pelo `hash` e exibe: nome do membro, título do produto, data de emissão e status "Certificado Válido".
3. **AC3** — Se o hash não for encontrado, exibe página com mensagem "Certificado não encontrado" e status "Inválido".
4. **AC4** — A página não requer login — é 100% pública para permitir compartilhamento e validação externa.
5. **AC5** — O endpoint `GET /api/certificates/validate/[hash]` retorna `{ valid: true, certificate: { memberName, productTitle, issuedAt } }` ou `{ valid: false }`.
6. **AC6** — A rota `/certificates` é adicionada ao `PUBLIC_ROUTES` no middleware para bypass de autenticação.
7. **AC7** — A página segue o design system Netflix-style (dark theme) com badge visual de "Certificado Válido" ou "Certificado Inválido".
8. **AC8** — A página inclui link para download do PDF se o certificado for válido.

### API Route — Validation Endpoint

```typescript
// src/app/api/certificates/validate/[hash]/route.ts
import { createAdminClient } from '@/lib/supabase/admin';
import { apiError, apiSuccess } from '@/lib/utils/api-response';

export async function GET(
  _request: NextRequest,
  { params }: { params: Promise<{ hash: string }> }
) {
  const { hash } = await params;
  const supabase = createAdminClient();

  const { data: certificate } = await supabase
    .from('certificates')
    .select('id, hash, issued_at, certificate_url, profiles(full_name), products(title)')
    .eq('hash', hash)
    .maybeSingle();

  if (!certificate) {
    return apiSuccess({ valid: false });
  }

  return apiSuccess({
    valid: true,
    certificate: {
      memberName: certificate.profiles?.full_name ?? 'Membro',
      productTitle: certificate.products?.title ?? 'Produto',
      issuedAt: certificate.issued_at,
      certificateUrl: certificate.certificate_url,
    },
  });
}
```

### Public Page — Server Component

A página deve usar `createAdminClient()` (service role) para query direta sem autenticação, pois é uma rota pública e o Supabase server client (cookie-based) não terá sessão.

```typescript
// src/app/certificates/[hash]/page.tsx
import { createAdminClient } from '@/lib/supabase/admin';

export default async function CertificateValidationPage({
  params,
}: {
  params: Promise<{ hash: string }>;
}) {
  const { hash } = await params;
  const supabase = createAdminClient();

  const { data: certificate } = await supabase
    .from('certificates')
    .select('id, hash, issued_at, certificate_url, profiles(full_name), products(title)')
    .eq('hash', hash)
    .maybeSingle();

  if (!certificate) {
    return <InvalidCertificate />;
  }

  return <ValidCertificate certificate={certificate} />;
}
```

### Page Design — Netflix-style

**Válido:**
- Background: `#141414`
- Card central com borda `#e50914` (primary)
- Badge verde "Certificado Válido" com ícone de check
- Nome do membro, produto, data em tipografia clara
- Botão "Baixar PDF" com link ao `certificate_url`

**Inválido:**
- Mesmo layout, mas badge vermelha "Certificado Não Encontrado"
- Mensagem: "O certificado com este código de validação não foi encontrado."

### Edge Cases

- **Hash inexistente:** Exibir página de "não encontrado" sem revelar informação sobre hashes válidos.
- **Certificate_url nulo:** Se o PDF ainda não foi uploaded (edge case improvável), não exibir botão de download.
- **SEO:** Adicionar `metadata` com `title` dinâmico para compartilhamento (ex: "Certificado de [Nome] - Memberly").
- **Rate limiting:** Considerar rate limit futuro para evitar enumeração de hashes (fora do escopo desta story).

### Testing

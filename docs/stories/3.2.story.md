## Story

**As a** membro,
**I want** ver todos os módulos de um produto com seu progresso,
**so that** eu saiba o que já assisti e o que falta.

---

## Acceptance Criteria

1. Página `/products/[slug]` exibe: banner hero do produto, título, descrição
2. Lista de módulos em grid ou lista vertical com: banner, título, descrição, número de aulas, progresso (X/Y concluídas)
3. Módulo clicável leva à página de aulas
4. Módulos exibidos na ordem definida pelo admin (`sort_order`)
5. Indicador visual de módulo concluído (100% das aulas assistidas)
6. Botão "Continuar de onde parei" que leva à próxima aula não concluída
7. Verificação de acesso: se membro não tem acesso ao produto, redirect para home com mensagem

### Supabase Query Patterns [Source: architecture.md#8-database-schema]

**Fetch produto com módulos e progresso (RSC — server-side):**
```typescript
const supabase = await createServerSupabaseClient();
const { data: { user } } = await supabase.auth.getUser();

// Produto por slug (RLS filtra acesso)
const { data: product } = await supabase
  .from('products')
  .select(`
    id, title, description, banner_url, slug,
    modules (
      id, title, description, banner_url, sort_order,
      lessons ( id, title, sort_order )
    )
  `)
  .eq('slug', params.slug)
  .eq('is_published', true)
  .single();

// Progresso do membro (todas as aulas concluídas)
const { data: progress } = await supabase
  .from('lesson_progress')
  .select('lesson_id, completed')
  .eq('profile_id', user.id)
  .eq('completed', true);

// Calcular progresso por módulo no servidor
const modulesWithProgress = product.modules
  .sort((a, b) => a.sort_order - b.sort_order)
  .map(mod => {
    const lessonIds = mod.lessons.map(l => l.id);
    const completedCount = progress.filter(p => lessonIds.includes(p.lesson_id)).length;
    return { ...mod, completedCount, totalLessons: lessonIds.length };
  });
```

**Encontrar próxima aula não concluída:**
```typescript
const completedLessonIds = new Set(progress.map(p => p.lesson_id));
const allLessons = product.modules
  .sort((a, b) => a.sort_order - b.sort_order)
  .flatMap(mod => mod.lessons.sort((a, b) => a.sort_order - b.sort_order));
const nextLesson = allLessons.find(l => !completedLessonIds.has(l.id));
```

### Accessibility [Source: front-end-spec.md#7-accessibility-requirements]

- Alt text em banners de módulo e produto
- Headings hierárquicos: h1 (produto), h2 ("Módulos"), h3 (nome do módulo)
- Cards como `<article>` com `aria-label` descritivo
- Focus indicators visíveis em cards e botões
- Progress bar com `role="progressbar"` + `aria-valuenow`
- "Continuar de onde parei": foco visual claro, botão grande (>= 44px)
